# 3. Day4：人工噪声下的 (z_p/z_n) split

## 做了什么

我做了一个最小版 split model：

```text
ECG x
  ↓
CNN Encoder
  ↓
shared z
  ├── z_p → disease classifier
  └── z_n → noise classifier
```

训练时先对 ECG 加人工噪声，再过 encoder。

每条样本有两个标签：

```text
disease label: 原始 ECG 疾病标签，不随人工噪声改变
noise label: clean / baseline_wander / gaussian / muscle / lead_dropout / amplitude_scaling
```

Loss：

[  
L = L_{cls} + \lambda L_{noise}  
]

其中：

```text
L_cls   = BCEWithLogitsLoss
L_noise = CrossEntropyLoss
```

我跑了：

```text
λ = 0.05
λ = 0.10
λ = 0.20
```

## 输出了什么

每个 run 输出：

```text

Noise confusion matrix
```

|---|--:|--:|--:|--:|--:|
|Split λ=0.05|0.9040|0.7596|0.6691|0.6853|0.6803|
|Split λ=0.10|0.9012|0.7594|0.6686|0.9047|0.9020|
|Split λ=0.20|0.8917|0.7373|0.6588|0.9409|0.9424|

最平衡的是：

```text
λ = 0.10
Disease AUROC = 0.9012
Noise Acc     = 0.9047
Noise Macro-F1 = 0.9020
```

Confusion matrix 也比较合理：

```text
clean:             309 / 325 correct
baseline_wander:   373 / 380 correct
gaussian:          365 / 371 correct
muscle:            358 / 384 correct
lead_dropout:      308 / 358 correct

```

主要混淆：


## 结论

> 在人工可控噪声下，(z_n) 确实可以学到噪声类型。但 λ 越大，noise 分类越强，disease 分类会下降。因此 λ=0.1 是人工噪声 split 的平衡点。

给导师可以说：

> Day4 回答了一个前置问题：如果噪声标签是可控的，(z_n) 是有能力学习噪声状态的。但这还是人工噪声，比较规则，所以我没有把它作为主实验，而是作为 sanity check。

# 7. Day6-A/B：Prototype Adapter 初版

## 做了什么

我尝试加入 APT-style prototype adapter，不加 Koopman。

两个版本：

### Day6-A：concat α

```text
z_n → α
concat(z_p, α) → disease classifier
```

### Day6-B：affine modulation

```text
z_n → α
α → γ, β
z_p' = γ · z_p + β
z_p' → disease classifier
```

训练策略沿用 Day5-B：

[  
L = L_{cls} + 0.2L_{quality} + 0.05L_{cons}  
]

## 输出了什么

输出：

```text

Quality metrics
Subgroup metrics
Alpha entropy
Prototype usage
Group-wise mean α
```

## 结果

Overall 没有超过 Day5-B：


但是 concat 在 baseline_drift subgroup 上有局部提升：

```text

AUROC +0.0066
AUPRC +0.0347
F1    +0.0330
```

## Alpha 分析发现

非 stopgrad concat 的 prototype 有明显 disease leakage。

Usage：

```text
proto0 = 55.4%
proto1 = 0.0%
proto2 = 0.0%
proto3 = 44.6%
```

Quality group 差异很小：

```text
clean_like:        α0=0.555, α3=0.444
```


> 直接 prototype adapter 存在 label leakage。虽然在 baseline_drift subgroup 上有局部收益，但 α 主要按 disease label 分，而不是按 quality/noise 分。因此不能直接把这个 α 用来做 noise-conditioned Koopman。

给导师可以说：

> 这个结果提醒我们，APT-style prototype 不能直接套用到 ECG 上。如果不加约束，prototype 很容易被 disease supervision 拉成 disease prototype。

# 8. Day6-C：Stop-gradient Prototype

## 做了什么

为减少 disease loss 对 α 的影响，我改成：

```python
alpha = prototype_head(z_n)
disease_logits = classifier(concat(z_p, alpha.detach()))
```

也就是说：

```text
quality loss 可以更新 z_n / prototype branch
disease loss 不能通过 α 更新 prototype branch
```

做了：

```text
p=4
p=2
```

## 输出了什么

输出：

```text
overall disease metrics
quality metrics
alpha usage
quality group mean α
disease group mean α
subgroup metrics
```

## 结果

Overall 仍然没有超过 Day5-B：

```text
Day5-B seed42 test AUROC = 0.9089
stopgrad p=4 test AUROC = 0.9063
stopgrad p=2 test AUROC = 0.9077
```

但是 disease leakage 明显减弱。

之前非 stopgrad：

```text
NORM α0=0.794
STTC α3=0.869
```

stopgrad p=4 后：

```text
NORM: α0=0.362, α1=0.354

STTC: α0=0.427, α1=0.288
CD:   α0=0.449, α1=0.276

```

说明 disease group 差异小了很多。

## 结论

> Stop-gradient 可以减少 disease leakage，并保留部分 subgroup 收益，但 α 与 quality group 的对齐仍然弱。因此 prototype 还不能说是成功的 quality/noise prototype。

# 9. Day6-D：Quality-bottleneck Prototype

## 做了什么

为了强制 α 学 quality，我进一步改成：

```text
z_n → α
α → quality classifier
concat(z_p, stopgrad(α)) → disease classifier
```

也就是 quality prediction 必须经过 α。

跑了：

```text
τ = 1.0
τ = 2.0
```

## 输出了什么

输出：

```text
disease metrics
quality metrics
alpha usage
alpha entropy
```

## 结果

发生完全 collapse。

τ=1：

```text
proto0 = 0
proto1 = 0
proto2 = 0
proto3 = 2198
mean entropy ≈ 0.000008
quality macro AUROC = 0.5074
```

τ=2：

```text
proto0 = 0
proto1 = 2198
proto2 = 0
proto3 = 0
mean entropy ≈ 0.000002
quality macro AUROC = 0.5004
```

## 结论

> Quality bottleneck 单独使用失败，所有样本 collapse 到一个 prototype，quality prediction 接近随机。Temperature 不能防止 collapse，因为模型可以放大 logits 重新变成 one-hot。

给导师可以说：

> 这个结果说明离散 prototype bottleneck 在没有 diversity / balancing 约束时不稳定。如果继续做 prototype，需要 very small diversity loss 或 class-balanced quality loss，否则不适合进入 Koopman。

# 10. 当前总结论

你可以把目前结论压成 4 句话：

## 结论 1：baseline 已经可靠

> CNN baseline 在 PTB-XL 上达到约 0.90 AUROC，多 seed 结果稳定，可以作为可靠对照。

## 结论 2：人工噪声证明 (z_n) 有基本能力

> 在可控人工噪声下，(z_n) 可以很好地区分噪声类型，λ=0.1 时 noise accuracy 达到 0.9047，同时 disease AUROC 只小幅下降。

## 结论 3：真实噪声主结果是 Day5-B

> Real-noise weak supervision + (z_p) consistency 是目前最稳的主结果。它在 3 个 seed 下小幅提升 disease classification，尤其 F1 提升明显，同时 (z_n) 对真实 quality metadata 有可学习信号。

## 结论 4：Prototype 目前还不成熟

> Prototype adapter 暴露了两个问题：直接训练会出现 disease leakage；强制 quality bottleneck 又会 collapse。因此目前还不能用 α 条件化 Koopman，需要进一步修 prototype，或者暂时回到 Day5-B 主线。

> 最后我尝试了 prototype adapter。结果发现直接 prototype 会出现 label leakage，α 主要按 disease 分；stop-gradient 可以缓解 disease leakage，但 quality alignment 仍然弱；强制 α 作为 quality bottleneck 又会发生 prototype collapse。因此我现在认为 prototype 还不成熟，暂时不能进入 Koopman。当前最稳主线应该是 Day5-B，后续如果继续做 prototype，需要加很小的 diversity / balancing 约束，或者重新设计 softer prototype。
