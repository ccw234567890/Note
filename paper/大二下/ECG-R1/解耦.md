### 解耦的双编码器架构（Decoupled Modalities Encoding）详解

#### 问题背景：传统"耦合"架构的缺陷

在 ECG-R1 之前，已有的 ECG 多模态大模型（如 GEM）采用**耦合编码策略（Coupled Encoding）**，如下图所示：

```
传统耦合架构（GEM）：
时序信号 → 时序编码器 → TS-Image对齐层 → 共享的Image-Language投影器 → <image>位置注入LLM
ECG图像 → 图像编码器 ──────────────────────────↑
```

这种设计存在两个严重问题：

1. **模态强耦合**：时序（TS）嵌入必须先经过一个 TS→Image 对齐层，然后**复用**图像-语言投影器，最终注入到 `<image>` token 位置。这意味着模型**默认两种模态总是同时出现**，单模态推理时很不自然。
2. **共享容量瓶颈**：一个投影器要同时处理两种异构模态（图像是二维渲染图，时序是一维信号），造成**表征妥协**——投影器无法同时为两种模态都学到最优映射。

#### ECG-R1 的解耦方案

ECG-R1 采用**解耦模态编码（Decoupled Modalities Encoding）**，核心思想是：**为每种模态配备独立的编码器和独立的投影器，并在 LLM 中使用不同的占位 token 注入**。

```
ECG-R1 解耦架构：
时序信号 → 时序编码器(ECG-CoCa) → 独立投影器Proj_T → <ecg>位置注入LLM
ECG图像 → 图像编码器(Qwen3-VL视觉部分) → 独立投影器Proj_I → <image>位置注入LLM
```

具体来说：

**① 两个独立的编码器：**

- **图像编码器**：使用 Qwen3-VL-8B 的视觉编码器，处理 ECG 渲染图（12导联波形图）
- **时序编码器**：使用 ECG-CoCa，处理原始 ECG 时间序列信号（一维数值序列）

**② 两个独立的投影器：**

- **Proj_I**：将图像嵌入映射到 LLM 的共享语义空间
- **Proj_T**：将时序嵌入映射到 LLM 的共享语义空间

两者互不共享参数，各自学习最适合自身模态的映射方式。

**③ 两个独立的占位 token：**

- 引入显式的 **`<ecg>`** 标签（放在 `<image>` 之前）
- 时序 token 块 **只注入到 `<ecg>` 位置**
- 图像 token 块 **只注入到 `<image>` 位置**

#### 数学表示

公式（1）给出了编码过程：

$$e_T = \text{Encoder}_T(x_T; \theta_{E_T}), \quad z_T = \text{Proj}_T(e_T; \theta_{\text{Proj}_T})$$

$$e_I = \text{Encoder}_I(x_I; \theta_{E_I}), \quad z_I = \text{Proj}_I(e_I; \theta_{\text{Proj}_I})$$

其中 $z_T, z_I \in \mathbb{R}^{\cdot \times d}$ 分别是时序和图像的嵌入向量，LLM 在 $x_{\text{text}}$（指令文本）的基础上，将 $z_T$ 注入到 `<ecg>` 位置、$z_I$ 注入到 `<image>` 位置。

#### 解耦带来的关键优势

|方面|耦合架构（GEM）|解耦架构（ECG-R1）|
|---|---|---|
|模态独立性|❌ 时序需经过图像对齐层|✅ 完全独立编码|
|投影器容量|❌ 共享瓶颈，表征妥协|✅ 各模态专用投影器|
|单模态推理|❌ 不自然，需填充占位|✅ 天然支持，可丢弃任一模态|
|与 IMD 配合|❌ 固定拼接，无法灵活丢弃|✅ 可独立丢弃任一模态的 token 块|

**核心洞见**：解耦设计使得 ECG-R1 在训练时可以灵活地通过 IMD 随机丢弃某个模态或交换 token 顺序，因为两种模态的编码路径完全独立，互不依赖。这正是 ECG-R1 实现**模态无关（modality-agnostic）**的基础——无论输入是 ECG 图像、时序信号、还是两者都有，模型都能可靠解读。
