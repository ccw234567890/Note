# $π_{0.7}$: a Steerable Generalist Robotic Foundation Model with Emergent Capabilities

## 世界模型与流匹配损失详解

### 一、世界模型（World Model）是什么？

在 π₀.₇ 中，**世界模型**是一个独立的**图像生成模型**，用于**实时生成子目标图像（Subgoal Images）**。它的核心功能是：给定当前观测和子任务指令，预测"近未来"的场景图像。

> "At runtime, the subgoal images are produced by a **lightweight world model**, which takes in the same subtask instruction $\hat{l}_t$ as the main model, but benefits from web-scale pre-training on videos and image editing tasks."
> (Intelligence 等, 2026)

#### 世界模型的架构

| 属性 | 值 |
|------|-----|
| **基础模型** | **BAGEL**（14B 参数的 Mixture-of-Transformers 模型） |
| **参数量** | **14B**（比 π₀.₇ 主模型 5B 还大） |
| **预训练数据** | 网络规模的视频和图像编辑任务 |
| **训练目标** | **流匹配（Flow Matching）损失** |
| **输入** | 当前观测 $o_t$ + 子任务指令 $\hat{l}_t$ + 片段元数据 $m$ |
| **输出** | 多视角子目标图像 $g_t = [G_t^1, ..., G_t^n]$ |

### 二、世界模型的训练目标

世界模型 $g_\psi$ 的训练目标：

$$
\max_\psi \mathbb{E}_{\mathcal{D}_g} \left[ \mathcal{L}_{\text{CFM}}\left( g^\star_t,\ g_\psi(o_t, \hat{l}_t, m) \right) \right]
$$

其中：
- $\mathcal{L}_{\text{CFM}}$：**标准流匹配损失（Standard Flow Matching Loss）**
- $g^\star_t$：**真实子目标图像**（训练轨迹中段末帧 $o_{t_{\text{end}}}$）
- $\mathcal{D}_g$：高质量子任务标注的数据子集
- $m$：片段元数据（速度、质量、错误标记）

#### 训练数据来源

> "By augmenting our world model training with web data, non-robot data sources such as egocentric human videos, and other video data, we can acquire semantic and physical concepts from these other data sources and then transfer them into π0.7 via subgoal images."
> (Intelligence 等, 2026)

世界模型不仅使用机器人数据训练，还混合了**网络数据、自我中心人类视频**等非机器人数据源，从而继承丰富的视觉语义知识和物理常识。

### 三、流匹配损失（Flow Matching Loss）详解

#### 3.1 核心概念

**流匹配（Flow Matching）** 是一种**生成式建模方法**，用于学习如何从**随机噪声**逐步**变换**到**目标数据分布**。可以理解为**扩散模型（Diffusion）的改进版本**。

> "The action expert typically uses a flow matching [102] (or diffusion) objective that captures the multi-modality of the robot actions."
> (Intelligence 等, 2026)

#### 3.2 为什么需要流匹配？—— 多模态性

机器人动作具有**多模态性（multi-modality）**：同一个任务可以有多种不同的执行方式。例如"抓杯子"可以从左边、右边、快或慢地伸手。传统的回归损失（如 MSE）会**平均**所有这些模式，导致动作模糊不精确。流匹配可以**保留多种可能的执行方式**。

#### 3.3 流匹配的数学原理

流匹配的核心思想是：**学习一个从噪声到数据的平滑变换路径（flow）**。

**基本流程**：
1. **起点**：一个简单的分布 $p_0$（如标准高斯噪声）
2. **终点**：目标数据分布 $p_1$（如真实的机器人动作分布）
3. **路径**：定义一个时间 $t \in [0,1]$ 上的概率路径，从 $p_0$ 平滑过渡到 $p_1$
4. **学习目标**：学习一个**向量场（vector field）** $v_\theta(x, t)$，它告诉每个点在每个时间步应该往哪个方向移动

**流匹配损失函数** $\mathcal{L}_{\text{CFM}}$：

$$
\mathcal{L}_{\text{CFM}} = \mathbb{E}_{t, x_0, x_1} \left[ \| v_\theta(x_t, t) - u_t(x_t | x_1) \|^2 \right]
$$

其中：
- $t \sim \mathcal{U}[0,1]$：随机采样时间步
- $x_0 \sim p_0$：从噪声分布采样
- $x_1 \sim p_1$：从真实数据分布采样（即训练数据中的真实动作/图像）
- $x_t = (1-t) \cdot x_0 + t \cdot x_1$：在噪声和真实数据之间做**线性插值**
- $u_t(x_t | x_1) = x_1 - x_0$：**目标向量场**——从噪声指向真实数据的方向
- $v_\theta(x_t, t)$：**模型预测的向量场**

**直观理解**：模型学习的是"给定当前状态 $x_t$ 和时间 $t$，应该往哪个方向走才能到达真实数据 $x_1$"。

#### 3.4 流匹配 vs 扩散模型

| 方面 | 扩散模型（Diffusion） | 流匹配（Flow Matching） |
|------|---------------------|----------------------|
| **路径** | 随机（通过随机微分方程） | **确定性**（通过常微分方程） |
| **采样步数** | 通常需要很多步（50-1000） | **更少步数**即可高质量采样 |
| **训练效率** | 需要预测噪声 | **直接预测向量场**，更简单 |
| **推理速度** | 较慢 | **更快**（适合机器人实时控制） |

### 四、流匹配在 π₀.₇ 中的具体应用

#### 4.1 动作专家（Action Expert）的流匹配

动作专家是一个 **860M 参数的 Transformer**，使用流匹配目标训练：

> "The more lightweight 'action expert' is a 860M-parameter transformer that is trained to predict continuous actions using flow matching objective."
> (Intelligence 等, 2026)

**训练过程**：
1. 从训练数据中取一个真实的动作块 $a_{t:t+H}$（50步连续动作）
2. 从高斯噪声中采样一个随机噪声 $a_0$
3. 在噪声和真实动作之间做线性插值：$a_\tau = (1-\tau) \cdot a_0 + \tau \cdot a_{t:t+H}$
4. 模型预测向量场 $v_\theta(a_\tau, \tau)$
5. 损失函数：预测的向量场与真实方向 $a_{t:t+H} - a_0$ 之间的 MSE

**推理过程**：
1. 从高斯噪声 $a_0$ 开始
2. 使用训练好的向量场 $v_\theta$，通过常微分方程（ODE）逐步求解
3. 经过若干步迭代（论文使用 5 步去噪），得到最终的动作块 $a_{t:t+H}$

#### 4.2 世界模型（World Model）的流匹配

世界模型同样使用流匹配损失训练，用于**图像生成**——从噪声生成子目标图像。

世界模型的训练数据采样策略（第 VI-C 节）：

> "We train on a combination of **real images from future timesteps of the training trajectory** and generated images."
> (Intelligence 等, 2026)

- **25% 概率**：采样段末帧（end-of-segment images），即子任务完成时的最后一帧
- **75% 概率**：采样 0-4 秒范围内的随机未来帧
- **额外混合**：世界模型自身生成的图像，用于弥合训练-推理差距

### 五、世界模型在推理时的工作流程

```
推理时的完整流程（Algorithm 1）：

1. 高层策略生成子任务指令 l̂_t
   （如 "grasp the handle"）

2. 世界模型 g_ψ 根据当前观测 o_t + 子任务 l̂_t + 元数据 m
   生成多视角子目标图像 g_t

3. π₀.₇ VLA 主模型接收：
   - 当前观测 o_t
   - 子任务指令 l̂_t
   - 子目标图像 g_t（由世界模型生成）
   - 元数据 m
   - 控制模式 c

4. π₀.₇ 生成动作块 a_{t:t+H}

5. 每 Δ=4 秒或子任务变化时，刷新子目标图像
```

### 六、世界模型 vs 动作专家：角色对比

| 角色 | 世界模型（World Model） | 动作专家（Action Expert） |
|------|----------------------|------------------------|
| **输入** | 当前观测 + 子任务 + 元数据 | VLM 骨干的激活值 |
| **输出** | **子目标图像**（视觉目标） | **动作块**（50步连续动作） |
| **参数量** | 14B（BAGEL） | 860M |
| **训练目标** | 流匹配（图像生成） | 流匹配（动作预测） |
| **功能** | "想象未来的场景应该是什么样子" | "计算如何从当前状态到达目标状态" |

### 七、直观理解

可以把世界模型理解为机器人的**"想象力"**：

- **人类做菜时**：你会在脑海中想象"下一步锅里的菜应该是什么样子"，然后根据这个想象调整动作
- **π₀.₇ 的世界模型**：做同样的事——根据当前画面和"下一步要做什么"的指令，**生成一张"未来应该长什么样"的图片**
- **π₀.₇ 的主模型**：看到当前画面和这张"未来图片"，计算出**需要执行什么动作**才能从"当前"到达"未来"

**流匹配的类比**：

> 流匹配就像**雕塑家**——从一块**粗糙的石头（噪声）** 开始，通过一系列**精细的雕刻步骤（ODE 求解）**，最终雕出一座**精美的雕像（真实动作/图像）**。而 MSE 回归就像**用 3D 扫描仪扫描所有雕像后取平均**——结果是一个模糊的、没有细节的轮廓。
