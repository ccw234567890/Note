# $π_{0.7}$: a Steerable Generalist Robotic Foundation Model with Emergent Capabilities

## 子任务指令生成来源

### 一、子任务指令的两种来源

论文原文（Section V-A）：

> "During inference, l̂t may be produced by a learned high-level policy or a human (or be omitted) and may change over time."

推理时，子任务指令 $l̂_t$ 可以来自：

| 来源 | 说明 | 使用场景 |
|------|------|---------|
| **人类实时提供（语言教练）** | 人类观察机器人执行过程，实时说出下一步指令 | 探索新任务时，人类一步步"教练"机器人 |
| **学习得到的高层策略** | 一个基于 π₀.₇ 架构微调的策略模型 | 收集教练数据后，训练成自主系统 |

### 二、高层策略是什么？是 LLM 吗？

论文原文：

> "After coaching we can take the verbal coaching data to finetune π0.7 as a high-level policy that maps the robot observations, task specification, and history of past subtask instructions to the new subtask instruction (Fig. 2 bottom left)."

**高层策略不是独立的 LLM（如 GPT-4），而是基于 π₀.₇ 自身架构微调得到的模型。**

具体来说：

```
高层策略 = π₀.₇ 微调版
输入：机器人观测 + 任务描述 + 历史子任务指令
输出：下一个子任务指令（文本）
```

**为什么不用现成的 LLM？** 因为高层策略需要：
1. **理解机器人当前的视觉观测**（摄像头画面）
2. **理解任务上下文**（已完成哪些子任务、还剩哪些）
3. **生成与机器人能力匹配的指令**（不能生成机器人做不到的指令）

### 三、完整的"教练→自主"流程

```
阶段 1：语言教练（人类提供子任务指令）
┌─────────────────────────────────────────────┐
│ 人类："打开冰箱门"                            │
│ 人类："取出食物"                              │
│ 人类："把食物放进微波炉"                       │
│ 人类："关上微波炉门"                          │
│         ↓ π₀.₇ 执行每个子任务                  │
│ 收集到：观测 + 子任务指令 的配对数据             │
└─────────────────────────────────────────────┘
                      ↓
阶段 2：训练高层策略
┌─────────────────────────────────────────────┐
│ 用收集到的教练数据微调 π₀.₇                    │
│ 输入：当前观测 + 任务描述 + 历史指令            │
│ 输出：下一个子任务指令                          │
└─────────────────────────────────────────────┘
                      ↓
阶段 3：完全自主执行
┌─────────────────────────────────────────────┐
│ 高层策略 → 生成子任务指令                      │
│         ↓                                    │
│ π₀.₇ 主模型 → 执行动作                        │
│         ↓                                    │
│ 循环直到任务完成                               │
└─────────────────────────────────────────────┘
```

### 四、训练时的子任务指令来源

训练时，子任务指令 $l̂_t$ 来自**人工标注**：

> "We collect data from a diverse set of tasks and scenarios, and then annotate the segments with detailed textual descriptions."

人类标注员将每个任务片段标注上对应的子任务描述（如"打开冰箱门"、"拿起削皮器"等）。

### 五、总结

| 问题 | 答案 |
|------|------|
| **高层策略是 LLM 吗？** | **不是独立的 LLM。** 它是基于 π₀.₇ 自身架构微调得到的策略模型 |
| **子任务指令由谁生成？** | 两种方式：**人类实时提供**（语言教练）或**高层策略自动生成**（自主模式） |
| **高层策略的输入是什么？** | 机器人观测 + 任务描述 + 历史子任务指令 |
| **高层策略的输出是什么？** | 下一个子任务指令（文本） |
| **训练数据从哪来？** | 人类"教练"机器人时收集的观测-指令配对数据 |
| **为什么不用现成 LLM？** | 高层策略需要理解机器人视觉观测和任务上下文，与 π₀.₇ 共享视觉编码器更高效 |
