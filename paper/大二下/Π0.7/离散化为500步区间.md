
# $π_{0.7}$: a Steerable Generalist Robotic Foundation Model with Emergent Capabilities

## "离散化为500步区间"详解

### 论文原文

> "Overall speed: the length of the episode in timesteps. We discretize the values in an interval of 500 steps, i.e., values between 1750 and 2250 are binned to '2000 steps'."

(Intelligence 等, 2026)

### 一、什么是"时间步（timestep）"？

机器人以固定频率运行（50Hz 或 20Hz），每个时间步就是机器人发出一次动作指令的时刻。例如：
- 50Hz 机器人：1秒 = 50个时间步
- 一个耗时 40 秒的任务 ≈ 2000 个时间步

### 二、"离散化为500步区间"的含义

就是把**连续的数值**（如 1873 步、2012 步、2234 步）**归入到以500为间隔的桶（bin）中**，变成一个**离散的类别标签**。

**具体规则**：

| 实际时间步数 | 归入的桶（bin） | 最终标签 |
|------------|---------------|---------|
| 0 ~ 500 | 0 ~ 500 → 取中间值 | "250 steps" |
| 501 ~ 1500 | 500 ~ 1500 → 取中间值 | "1000 steps" |
| 1501 ~ 2500 | 1500 ~ 2500 → 取中间值 | "2000 steps" |
| 2501 ~ 3500 | 2500 ~ 3500 → 取中间值 | "3000 steps" |
| ... | ... | ... |

**论文中的例子**：
- 实际时间步数在 **1750 ~ 2250** 之间 → 归入 **"2000 steps"** 这个桶

### 三、"8000 steps" 的含义

"8000 steps" 意味着这个片段的实际时间步数在 **7750 ~ 8250** 之间（归入 8000 的桶）。

换算成实际时间（以 50Hz 机器人为例）：
- 8000 步 ÷ 50 步/秒 = **160 秒 ≈ 2 分 40 秒**

这是一个比较长的任务（比如"削果蔬"这种需要精细操作的任务）。

### 四、为什么要这样做？

| 原因 | 说明 |
|------|------|
| **避免过拟合** | 如果直接用精确数值（如 1873 步），模型可能记住"这个精确数字对应这个特定片段"，而不是学到"速度"这个抽象概念 |
| **泛化能力** | 离散化后，"2000 steps" 代表"大约 40 秒左右的任务"，模型能泛化到不同长度的同类任务 |
| **文本兼容** | 这些元数据以文本形式输入模型（如 `Speed: 2000. Quality: 5. Mistake: false.`），离散化后的文本更简洁稳定 |

### 五、直观类比

就像考试分数：你考了 **87 分**（精确值），但老师只告诉你等级：**"B 等"**（离散化后的类别）。你不需要知道精确的 87 分，只需要知道"这是中等偏上的水平"就够了。

### 六、总结

| 问题 | 答案 |
|------|------|
| **"离散化为500步区间"是什么意思？** | 把精确的时间步数（如 1873）归入以500为间隔的桶中，变成离散标签（如 "2000 steps"） |
| **"8000 steps" 代表什么？** | 实际时间步数在 7750~8250 之间，约 160 秒（50Hz）或 400 秒（20Hz） |
| **为什么这么做？** | 避免过拟合、增强泛化、使元数据以简洁文本形式输入模型 |
| **推理时怎么用？** | 用户直接告诉模型想要的速度，如 `Speed: 2000.` 让模型以"快速"模式执行 |
