# 6. Day5 的 noise label 怎么做？

可以把 PTB-XL 里的真实 noise 字段变成 multi-label：

```text
static_noise
burst_noise
baseline_drift
electrodes_problems
```

每条 ECG 可以有多个质量问题，所以这里不要用 CrossEntropy，而要用 BCE。

也就是：

```text
noise_target = [
  has_static_noise,
  has_burst_noise,
  has_baseline_drift,
  has_electrodes_problems
]
```

然后：

[  
L = L_{cls} + \lambda L_{real-noise}  
]

其中：

[  
L_{cls}=BCE(\hat{y}, y)  
]

[  
L_{real-noise}=BCE(\hat{q}, q)  
]

这里 (q) 是真实噪声/质量 metadata。

# 8. 但是导师说“直接 encoder 分太粗暴”，怎么办？

导师的意思是：不能只靠两个 MLP head，然后期待它自动分好。你需要一点训练策略。

但这个训练策略不要太复杂。

现在可以加一个最简单、最合理的：

## clean/noisy consistency

即使主线用真实噪声，也可以用轻度人工增强做 regularization。

对于同一条 ECG：

```text
x_original
x_augmented
```

要求：

[  
z_p(x_{original}) \approx z_p(x_{augmented})  
]

这表示：

> 疾病相关生理表征应该对扰动稳定。

同时 (z_n) 可以变化，因为噪声状态变了。

最小 loss：

[  
L_{cons}=||z_p(x)-z_p(\tilde{x})||^2  
]

这样总 loss 变成：

[  
L = L_{cls}+\lambda_1 L_{real-noise}+\lambda_2 L_{cons}  
]

这个比直接 MLP 分支更有策略，也不算复杂。
