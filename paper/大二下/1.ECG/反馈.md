# Day5-B 模型

结构不变：

```text
ECG x
  ↓
Encoder
  ↓
shared z
  ├── z_p → disease classifier
  └── z_n → quality classifier
```

但是训练时对同一条 ECG 构造一个轻度扰动版本：

```text
x          = 原始 ECG
x_aug      = 轻度人工扰动 ECG
```

然后要求：

[  
z_p(x) \approx z_p(x_{aug})  
]

也就是：

[  
L_{cons} = ||z_p(x)-z_p(x_{aug})||^2  
]

总 loss：

[  
L = L_{cls} + \lambda_q L_{quality} + \lambda_c L_{cons}  
]

建议先固定：

```text
λ_q = 0.2
```

因为 Day5-A 里 λ=0.2 对 quality 学得最好，同时 disease 只掉一点。

然后扫：

```text
λ_c = 0.01
λ_c = 0.05
λ_c = 0.10
```

```text

muscle level 1
amplitude_scaling level 1
```

Day5-B 不是为了训练人工噪声分类，而是为了让：

```text

z_n 捕捉真实 quality metadata
```

## 2. 为什么 smoke test 里 consistency 很小，但最后还能提升？

这很正常。原因有几个：

### 第一，consistency 不一定要很大才有效

Consistency loss 的作用不是主导训练，而是轻微约束 (z_p) 的方向：

[  
z_p(x) \approx z_p(x_{aug})  
]

即使数值小，它也可能在训练过程中起到 regularization 的作用，让 (z_p) 少学一些扰动相关信息。

也就是说，它的作用更像：

```text
轻量正则项
```

不是一个强监督任务。

---

### 第二，你后面正式训练的 cons 不再是 1e-5

日志里后期 cons 已经到：

```text
0.006 - 0.010
```

比如：

```text
epoch 019 cons = 0.008604
epoch 020 cons = 0.009113
```

所以它不是完全没信号。可能 smoke test 时模型还没训练起来，或者 batch/扰动组合导致数值特别小。

---

### 第三，consistency 可能提升了 disease representation 的稳定性

Day5-A 只是让：

```text
z_n → quality metadata
z_p → disease
```

但没有明确告诉模型：

```text
z_p 不应该对轻度扰动敏感
```

Day5-B 加了这个约束，所以更符合我们的分工：


这就解释了为什么 Day5-B 的 disease AUROC 反而更好。
