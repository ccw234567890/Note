# Transformer Hawkes Process

## 引入图结构是什么意思？

**图（Graph）** 是数学中表示**实体之间关系**的数据结构，由两部分组成：
- **顶点（Vertices/Nodes）**：实体本身
- **边（Edges）**：实体之间的关系

论文中使用的 **911-Calls 数据集**示例：
- 顶点 = 不同的城市/地区（如 Pittsburgh, Philadelphia, ...）
- 边 = 两个城市在地理上相邻

当一个城市发生紧急事件时，邻近城市也可能受到影响——这就是图结构要捕获的信息。

---

## 为什么需要引入图结构？

### 问题场景

假设有多个相关的事件序列：

| 序列 | 事件 | 描述 |
|:----:|:----:|:------|
| **Pittsburgh** | 911呼叫记录 | 城市A的紧急事件 |
| **Philadelphia** | 911呼叫记录 | 城市B的紧急事件 |
| **Erie** | 911呼叫记录 | 城市C的紧急事件 |

**传统方法（包括 THP 基础版）**：每个序列独立建模，Pittsburgh 的模型不知道 Philadelphia 发生了什么。

**现实情况**：如果 Pittsburgh 发生大地震，Philadelphia 的紧急呼叫量也会激增——因为邻近城市会受到影响。

**引入图结构**就是让模型知道："Pittsburgh 和 Philadelphia 是相邻的，它们的事件会互相影响。"

---

## 论文中具体怎么做的？

### 第一步：定义图

论文在 **911-Calls** 和 **Earthquake** 两个数据集上使用了图结构：

$$G = (V, E)$$

$$V = \{\text{城市1, 城市2, ..., 城市N}\} \quad \text{—— 顶点}$$

$$E = \{(\text{城市1, 城市2}), (\text{城市2, 城市3}), ...\} \quad \text{—— 边（地理相邻关系）}$$

### 第二步：顶点嵌入

每个城市被映射到一个向量：

$$E \in \mathbb{R}^{M \times |V|}$$

其中 $E(:, v)$ 是城市 $v$ 的 $M$ 维嵌入向量。例如：
- Pittsburgh → [0.23, -0.15, 0.67, ...]
- Philadelphia → [0.21, -0.12, 0.71, ...]（与 Pittsburgh 相似，因为相邻）
- Erie → [-0.45, 0.33, -0.12, ...]（与 Pittsburgh 不同，因为不相邻）

### 第三步：修改事件嵌入

原来的事件嵌入只包含事件类型 + 时间：

$$X = (UY + Z)^\top$$

引入图结构后，嵌入变为：

$$X = (UY + EV + Z)^\top$$

多了一项 $EV$，表示**这个事件发生在哪个城市**。

### 第四步：图注意力

这是最核心的改动。在自注意力计算中，加入一个**顶点相似度矩阵**：

$$S = \text{Softmax}\left(\frac{QK^\top}{\sqrt{M_K}} + A\right)V_{\text{value}}$$

其中 $A_{ij}$ 表示事件 $i$ 所在的城市和事件 $j$ 所在的城市之间的相似度：

$$A_{ij} = (EV_i)^\top \Omega (EV_j)$$

**直观理解**：
- 如果两个事件发生在**同一个城市**：$A_{ij}$ 为大正数 → 注意力权重增大
- 如果两个事件发生在**相邻城市**：$A_{ij}$ 为中等正数 → 注意力权重适度增大
- 如果两个事件发生在**不相邻城市**：$A_{ij}$ 接近0或负数 → 注意力权重不变或减小

### 第五步：图正则化

为了让顶点嵌入真正反映图结构，加入正则化损失：

$$\mathcal{L}_{\text{graph}} = \sum_{j,k} \left[-\log(1 + e^{V_j \Omega V_k}) + \mathbf{1}\{(v_j, v_k) \in E\}(V_j \Omega V_k)\right]$$

**直观理解**：
- 如果两个城市**有边相连**（相邻），鼓励它们的嵌入相似度高
- 如果两个城市**没有边相连**，鼓励它们的嵌入相似度低

## 一个具体的例子

以 **Earthquake 数据集**为例：

```
图结构：
  北京 ── 天津
   │       │
   │       │
  成都 ── 重庆

事件序列：
  北京: [t1: 地震3.2级, t5: 地震2.1级, ...]
  天津: [t2: 地震1.5级, t7: 地震4.0级, ...]
  成都: [t3: 地震2.8级, ...]
  重庆: [t4: 地震1.0级, ...]
```

**没有图结构**（普通 THP）：
- 预测天津 t7 的地震时，只能使用天津自己的历史
- 不知道北京 t5 的地震可能预示着天津 t7 的地震

**有图结构**（Structured-THP）：
- 预测天津 t7 的地震时，自注意力会**自动关注**北京 t5 的事件
- 因为北京和天津在图中有边相连，$A_{ij}$ 增大了它们之间的注意力权重
- 模型学到了："北京地震后，天津也容易地震"

## 多模态 ECG 时间序列的图结构应用

### 核心判断：可以，但需要关键适配

| THP 的能力 | ECG 的需求 | 匹配度 |
|-----------|-----------|:------:|
| 异步事件序列建模 | ECG 是**规则采样**的连续信号，不是离散事件 | ⚠️ 需要改造 |
| 自注意力捕获长程依赖 | ECG 中早搏与后续心律失常有长程关联 | ✅ 完美 |
| 时间编码感知时间间隔 | ECG 的 RR 间期（心跳间隔）本身是重要特征 | ✅ 适用 |
| 多头注意力捕获多种模式 | ECG 中 P波/QRS波/T波各有不同形态特征 | ✅ 适用 |
| 图结构融合多模态 | ECG + 血压 + SpO2 + 呼吸等多模态信号 | ✅ 核心优势 |

### 多模态融合方案

将**每个模态视为一个顶点**，构建多模态图：

```
图结构 G = (V, E):

V = {ECG, 血压(BP), 血氧(SpO2), 呼吸(Resp)}

E = {
  (ECG, BP),      ← 心电与血压强相关
  (ECG, SpO2),    ← 心电与血氧相关
  (BP, Resp),     ← 血压与呼吸相关
  (ECG, Resp),    ← 心电与呼吸相关
  ...
}
```

**图注意力机制**让模型自动学习跨模态的因果关系：
- ECG 的 ST 段抬高事件会**自动关注**同时刻的血压骤升事件
- SpO2 下降事件会**自动关注**之前的呼吸异常事件

### 临床场景示例：预测室颤（VF）

**输入**：ECG 导联 I、II、V5 + 血压 + 血氧饱和度（5个模态顶点）

```
事件序列:
  t1: ECG导联I → 室性早搏
  t2: 血压 → 收缩压下降
  t3: SpO2 → 血氧降至88%
  t4: ECG导联V5 → T波倒置
  t5: ECG导联II → ST段压低
  
图结构:
  ECG-I ── ECG-II ── ECG-V5
    │                    │
    └──── 血压 ──── SpO2 ┘
```

**预测**：$\lambda_{\text{室颤}}(t|\mathcal{H}_t)$ 持续升高 → 发出预警

---

## 一句话总结

**引入图结构 = 让模型知道"哪些序列之间是相关的"，并在建模时自动利用这种相关性来互相学习。**

就像人脑的联想记忆——当你听说"北京发生地震"时，你会自然想到"天津可能也会受影响"。Structured-THP 就是用数学方式让模型也具备这种联想能力。
