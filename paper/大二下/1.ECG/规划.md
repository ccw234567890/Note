我们现在应该**先做最小可验证 prototype**，不要一开始就把 APT、Koopman、(z_p/z_n)、residual、7 个 loss 全部堆上去。导师的意思很明确：**先用一个好 handle 的数据集，跑一个简单实验；模型要简洁，每次只加一个东西，看它到底有没有用**。 导师也提醒过，模型太复杂、loss 太多会很不稳定，最好一个 block 解决一个问题。

下面给你一个实际执行框架。

---

# 总目标

先不要急着做完整论文级模型。第一阶段目标是：

> **在 PTB-XL 上跑通一个最小 ECG 分类框架，然后逐步验证 (z_n)、prototype、Koopman 是否真的有用。**

也就是从：

```text
ECG → Encoder → Classifier
```

一步步加到：

```text
ECG → Encoder → z_p / z_n → α → K_α z_p → Classifier
```

# 第一阶段：先跑通 PTB-XL baseline

## 目标

先证明你能正常读取数据、训练模型、得到合理结果。

## 输入

用 PTB-XL 的 `records100`：

```text
X shape = [12, 1000]
```

标签先用 5 个 superclass：

```text
NORM, MI, STTC, CD, HYP
```

划分：

```text
train: fold 1-8
val: fold 9
test: fold 10
```

## 模型

先用最简单的：

```text
12-lead ECG
   ↓
1D CNN / ResNet1D Encoder
   ↓
Global Pooling
   ↓
MLP Classifier
   ↓
5-label sigmoid output
```

## Loss

只用：

[  
L_{cls}=BCE(\hat{y},y)  
]

## 你要得到的结果

至少要有：

```text
AUROC
AUPRC
F1
loss curve
```

这一阶段不要加 Koopman，不要加 APT，不要加 (z_p/z_n)。

# 第二阶段：加入人工噪声，验证 noise 问题是否真实存在

导师最关心的是 noisy composition，也就是噪声到底能不能被分出来。 所以第二步不是直接加 Koopman，而是先构造可控噪声。

## 人工噪声类型

给 PTB-XL 加几类噪声：

```text
baseline wander
Gaussian noise
muscle artifact / high-frequency noise
powerline noise
lead dropout
amplitude scaling
```

## 实验

训练还是原始 baseline，测试时加噪声：

```text
clean test
noise level 1
noise level 2
noise level 3
```

## 看什么

看不同噪声下性能下降多少：

```text
AUROC drop
F1 drop
worst-noise performance
```

这一步的意义是：

> 先证明噪声确实会破坏 ECG 分类，后面你的 noise prototype 才有必要。

---

# 第三阶段：加入 (z_p / z_n) 分支

这一步是你方法的核心前置验证。

## 模型

```text
ECG
 ↓
Encoder
 ↓
shared representation z
 ├── physiological head → z_p → classifier
 └── nuisance head → z_n → noise/domain predictor
```

## 目标

让：

```text
z_p 负责疾病分类
z_n 负责噪声/域信息
```

## Loss

先只加两个 loss：

[  
L = L_{cls} + \lambda L_{noise}  
]

其中：

- (L_{cls})：用 (z_p) 做疾病分类；
    
- (L_{noise})：用 (z_n) 预测人工噪声类型。
    

不要一开始就加 adversarial loss、orthogonal loss、prototype diversity loss。

## 你要验证什么？

这一阶段最重要的是验证：

> (z_n) 到底能不能学到噪声信息？

你可以做两个简单分析：

### 1. noise prediction accuracy

看 (z_n) 能不能预测：

```text
clean
baseline wander
Gaussian noise
lead dropout
powerline noise
```

### 2. UMAP / t-SNE

把 (z_n) 可视化，看不同噪声类型是否能分开。

如果 (z_n) 分不开噪声，那后面：

[  
z_n \rightarrow \alpha \rightarrow K_\alpha  
]

就很难成立。

---

# 第四阶段：加入 prototype，但先不要加 Koopman

这一步对应 APT 的改造版。

## 模型

```text
z_n
 ↓
prototype matching
 ↓
α
 ↓
feature modulation
 ↓
z_p'
 ↓
classifier
```

公式：

[  
\alpha = \text{softmax}(g(z_n))  
]

[  
\tilde{z}_p = \gamma_\alpha \odot z_p + \beta_\alpha  
]

[  
\hat{y}=C(\tilde{z}_p)  
]

## 目标

证明：

> 用 (z_n) 生成 prototype 权重 (\alpha)，确实能提升噪声下分类鲁棒性。

## 对比

```text
Baseline
z_p/z_n split only
z_n → α → affine modulation
```

如果 prototype modulation 没有提升，那就说明 APT 这部分暂时不 work，需要先改它，不要急着加 Koopman。

---

# 第五阶段：加入 fixed Koopman

这一步先不要上 (K_\alpha)，只用固定 (K)。

## 模型

```text
z_p,t
 ↓
K
 ↓
z_hat_p,t+1
 ↓
Koopman loss
```

公式：

[  
\hat{z}_{p,t+1}=Kz_{p,t}  
]

[  
L_{koop}=||z_{p,t+1}-Kz_{p,t}||^2  
]

总 loss：

[  
L = L_{cls} + \lambda L_{koop}  
]

## 目标

证明：

> Koopman dynamic consistency 对 ECG 分类有没有帮助。

## 对比

```text
Baseline
+ z_p/z_n split
+ fixed Koopman
```

如果 fixed Koopman 都没有任何提升，说明 Koopman motivation 需要重新想。

导师之前也问过：通过 ECG 一段预测下一段，这个 dynamic 到底表征什么？所以这一步非常关键。

---

# 第六阶段：最后再做 prototype-conditioned Koopman

如果前面几步都 work，再做你的核心方法：

[  
z_n \rightarrow \alpha  
]

[  
K_\alpha=\sum_i \alpha_i K_i  
]

[  
\hat{z}_{p,t+1}=K_\alpha z_{p,t}  
]

## 模型

```text
ECG
 ↓
Encoder
 ↓
z
 ├── z_p ───────────────┐
 │                      ↓
 └── z_n → prototype α → K_α
                        ↓
              z_hat_p,t+1
                        ↓
                  classifier
```

## 关键对比

```text
fixed K
vs
K_α = Σ α_i K_i
```

如果 (K_\alpha) 比 fixed (K) 好，你就能说明：

> 噪声/域 prototype 条件化 Koopman 是有用的。

# 最终框架一句话

你现在的执行框架应该是：

> **先用 PTB-XL 跑通 baseline；再用人工噪声验证噪声问题；然后训练 (z_p/z_n) 分支，看 (z_n) 能不能学到噪声；接着加入 prototype adapter；最后才加入 Koopman，并比较 fixed (K) 和 prototype-conditioned (K_\alpha)。**

最小主线是：

```text
PTB-XL
→ Baseline ECG classifier
→ Noisy robustness test
→ z_p / z_n split
→ z_n prototype adapter
→ fixed Koopman
→ prototype-conditioned Koopman
→ ablation table
```
