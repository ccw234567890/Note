### 期望末端姿态（Desired End-Effector Pose）

#### 一句话定义

**期望末端姿态**就是RL策略告诉机器人"你的手应该移动到什么位置、以什么角度摆放"的目标指令。

---

#### 拆解：姿态（Pose） = 位置（Position） + 朝向（Orientation）

末端执行器的完整状态由**6个自由度**描述：

```
末端姿态 = [x, y, z, roll, pitch, yaw]
           └──位置──┘  └────朝向────┘
           平移(3DoF)    旋转(3DoF)
```

|分量|含义|单位|
|---|---|---|
|**x, y, z**|手在三维空间中的位置|米（mm）|
|**roll (φ)**|绕x轴旋转（翻滚）|弧度/度|
|**pitch (θ)**|绕y轴旋转（俯仰）|弧度/度|
|**yaw (ψ)**|绕z轴旋转（偏航）|弧度/度|

---

#### 在SERL中的具体含义

SERL的策略输出的是**增量式期望末端姿态**（delta pose），即相对于当前姿态的微小变化：

> The action space is the 6 DoF delta pose of the end-effector.

(Luo 等, 2025)

```
RL策略输出: a = [Δx, Δy, Δz, Δroll, Δpitch, Δyaw]
                    ↓
期望末端姿态: p_ref = p_current + a
                    ↓
阻抗控制器跟踪 p_ref
```

**直观理解**：RL策略不是直接告诉机器人"手放到坐标(0.5, 0.3, 0.2)"，而是告诉它"手再往前移1cm，再顺时针转2度"——每次只做一个微小的调整。

---

#### 为什么是"期望"而不是"实际"？

因为**期望 ≠ 实际**。阻抗控制器会尽力让机器人到达 $p_{ref}$，但实际到达的位置 $p$ 可能不同：

```
期望末端姿态 p_ref: RL策略想要的位置
        ↓ 阻抗控制器尽力跟踪
实际末端姿态 p:     机器人真正到达的位置
        ↓
误差 e = p - p_ref: 两者之间的差距
```

**在接触时**，这个误差会被参考限幅器限制：

> $|e| \leq \Delta$

(Luo 等, 2025)

例如PCB插装时，RL说"再往下1cm"，但实际已经接触PCB板了，阻抗控制器会**限幅**，实际只往下走了几微米——这就是"期望"和"实际"的区别。

#### 在SERL控制层级中的位置

```
┌──────────────────────────────────────────────┐
│  RL策略 π(a|s)        @ 10 Hz                │
│  输出: 期望末端姿态 p_ref (6D delta pose)      │
│         ↓                                     │
│  每个RL步 = 100个控制步                         │
│         ↓                                     │
│  阻抗控制器           @ 1 KHz                  │
│  目标: 跟踪 p_ref                               │
│  公式: F = k_p·e + k_d·ė + F_ff + F_cor       │
│         ↓                                     │
│  机器人实际到达 p                               │
└──────────────────────────────────────────────┘
```

#### 总结

|概念|解释|
|---|---|
|**末端**|机器人手部（末端执行器）|
|**姿态**|位置(x,y,z) + 朝向(roll,pitch,yaw)|
|**期望**|RL策略想要达到的目标，不一定等于实际|
|**增量式**|每次输出相对于当前姿态的微小变化|
|**在SERL中**|策略输出6D delta pose → 阻抗控制器跟踪 → 机器人执行|
