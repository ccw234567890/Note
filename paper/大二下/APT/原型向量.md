## 原型向量（Prototype Vectors）是什么？

### 一、数学定义

论文公式(5)中明确给出了原型向量的定义：

> _"we match each timestamp embedding $T_t \in \mathbb{R}^D$ against a shared learnable prototype library $P = \{p_j \in \mathbb{R}^D\}^N_{j=1}$ via inner product similarity"_

**原型向量就是一组可学习的向量**：

$$P = \{p_1, p_2, p_3, ..., p_N\}, \quad p_j \in \mathbb{R}^D$$

其中：

|符号|含义|具体数值|
|---|---|---|
|$N$|原型数量|5~40（取决于数据集频率）|
|$D$|嵌入维度|**20**（论文默认值）|
|$p_j$|第 j 个原型向量|$mathbb{R}^{20}$ 维的可学习向量|

所以原型库 $P$ 本质上是一个**可训练的矩阵**：

$$P \in \mathbb{R}^{N \times D}$$

### 二、原型向量 vs 原始时间戳嵌入：本质区别

这是理解原型学习最关键的一点。我们对比一下：

#### 原始时间戳嵌入（不用原型）

每个时间戳组合有一个独立的嵌入向量：

```
Monday 08:00 → 嵌入向量 A ∈ ℝ²⁰
Monday 08:10 → 嵌入向量 B ∈ ℝ²⁰
Monday 08:20 → 嵌入向量 C ∈ ℝ²⁰
...           → ...
Sunday 23:50 → 嵌入向量 X ∈ ℝ²⁰
```

**问题**：最多有 **1008 个**不同的嵌入向量需要学习，每个只出现在少数样本中。

#### 原型向量（使用原型学习）

只有 **N 个**共享的原型向量（比如 N=40）：

```
原型向量 p₁ ∈ ℝ²⁰  → 代表"早高峰模式"
原型向量 p₂ ∈ ℝ²⁰  → 代表"晚高峰模式"
原型向量 p₃ ∈ ℝ²⁰  → 代表"夜间低谷模式"
...
原型向量 p₄₀ ∈ ℝ²⁰ → 代表"周末模式"
```

每个时间戳不再有自己的专属嵌入，而是**从这 40 个原型中选择最相似的几个来组合表示自己**。

### 三、原型向量的物理含义

论文通过可视化验证了原型向量的语义（图5）：

> _"Visualization of APT timestamps and prototype embeddings on ECL dataset and iTransformer"_

每个原型向量 $p_j$ 实际上编码了一种**典型的全局分布模式**：

|原型|可能的语义|对应的时间段|
|---|---|---|
|$p_1$|**早高峰模式**（高用电量、快速上升）|周一~周五 07:00-09:00|
|$p_2$|**工作时段模式**（平稳中等用电量）|周一~周五 09:00-17:00|
|$p_3$|**晚高峰模式**（高用电量、缓慢下降）|周一~周五 18:00-21:00|
|$p_4$|**夜间低谷模式**（低用电量、平稳）|每天 00:00-05:00|
|$p_5$|**周末白天模式**（中等偏低、波动小）|周六~周日 10:00-18:00|
|...|...|...|

这些原型向量是**从所有时间戳数据中自动学习出来的**，而不是人工指定的。

### 四、原型向量是如何被"使用"的？

#### 步骤1：匹配（公式5）

计算时间戳嵌入 $T_t$ 与每个原型向量 $p_j$ 的**内积相似度**：

$$S_t = T_t P^\top$$

这相当于问：**"当前时间戳 Monday 08:00，和哪个原型最像？"**

```
T_t · p₁ = 8.5  → 最像"早高峰模式" ✅
T_t · p₂ = 2.1  → 有点像"工作时段"
T_t · p₃ = -1.3 → 不像"晚高峰"
T_t · p₄ = -0.5 → 不像"夜间低谷"
T_t · p₅ = 6.2  → 也比较像"周末模式"
```

#### 步骤2：选择 Top-k（公式6）

只保留最相似的 k 个原型（k=2），做 Softmax：

|选中的原型|相似度|Softmax 权重|
|---|---|---|
|$p_1$（早高峰）|8.5|**0.91**|
|$p_5$（周末白天）|6.2|**0.09**|

#### 步骤3：加权聚合（公式7）

$$\tilde{T}_t = 0.91 \times p_1 + 0.09 \times p_5$$

最终的时间戳表示 $\tilde{T}_t$ 是原型向量的**加权组合**，而不是原始嵌入。

### 五、原型向量是怎么学出来的？

原型向量 $P$ 是**可训练参数**，通过两个阶段优化：

#### 阶段一：自监督预训练

冻结 backbone 和归一化模块，只优化 APT 的参数（包括原型向量 $P$），使用三个损失：

1. **正交损失** $\mathcal{L}_{orth}$：让原型向量之间彼此正交，保证多样性
2. **负载均衡损失** $\mathcal{L}_{balance}$：让所有原型被均匀使用，防止某些原型"饿死"
3. **仿射正则化损失** $\mathcal{L}_R$：稳定仿射参数

#### 阶段二：联合优化

APT 与 backbone 一起在 MSE 损失下微调，原型向量继续更新。

### 六、一个直观的类比

> **原型向量就像"调色板上的基础色"**

|概念|类比|
|---|---|
|**原型向量 $p_j$**|调色板上的**基础色**（红、黄、蓝、绿...）|
|**原始时间戳嵌入 $T_t$**|你想调出的**目标颜色**|
|**相似度计算 $S_t$**|判断目标颜色最接近哪几种基础色|
|**Top-k 选择**|只选最接近的 2-3 种基础色|
|**加权聚合 $tilde{T}_t$**|按比例混合基础色得到目标色|

**不用原型**：为每个目标颜色单独调色 → 1008 种颜色，每种只用几次，调不准

**用原型**：只学 40 种基础色，任何目标颜色都用基础色混合得到 → 基础色被反复使用，学得准

### 七、关键参数总结

|参数|值|含义|
|---|---|---|
|原型数量 $N$|5~40|数据集频率越高，原型越多|
|嵌入维度 $D$|**20**|每个原型向量的长度|
|Top-k|**2**|每个时间戳只选最相似的 2 个原型|
|原型库大小|$N times D$|40×20=800 个参数（极小！）|

### 一句话总结

> **原型向量就是一组可学习的"典型分布模式"向量。每个时间戳不再用自己的专属嵌入，而是从这组共享的原型中选出最相似的几个，用它们的加权组合来表示自己。这就像用调色板上的几十种基础色调出任意颜色——原型向量被所有时间戳共享使用，因此每个原型都能从大量数据中充分学习，解决了稀疏时间戳组合的少样本问题。**
