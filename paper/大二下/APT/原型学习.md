## 什么是原型学习（Prototype Learning）？

### 一、先理解问题：为什么需要原型？

论文把时间戳离散化为分类属性后，得到了每个时间戳的嵌入向量 $T_t$。但这里有一个问题：

**问题：时间戳组合可能非常稀疏**

以 ECL 数据集为例（每10分钟采样一次）：

|时间戳属性|可能的取值数量|
|---|---|
|TiD（一天中的时间）|144 种（24小时 × 6个10分钟）|
|DiW（一周中的天）|7 种|
|**组合总数**|**144 × 7 = 1,008 种**|

但训练数据中可能只覆盖了其中一部分组合。比如：

|时间戳组合|训练集中出现次数|
|---|---|
|Monday 08:00|50次 ✅ 充足|
|Monday 08:10|50次 ✅ 充足|
|...|...|
|**Sunday 23:50**|**可能只有2-3次** ❌ 稀疏|

对于出现次数很少的时间戳组合，模型学到的嵌入向量**不可靠**，泛化能力差。

### 二、原型学习的核心思想

> **不直接使用原始时间戳嵌入，而是从时间戳嵌入中提取"原型"——即最典型的分布模式。**

论文原文：

> _"Inspired by prototype learning in few-shot learning, we propose to learn a shared prototype library to replace the raw timestamp embeddings."_

#### 生活类比：认识"周一早上8点"的用电模式

假设你只有3个周一早上8点的数据样本：

```
样本1: [用电量=100, 102, 98, ...]
样本2: [用电量=105, 99, 101, ...]
样本3: [用电量=97, 103, 100, ...]
```

**直接学习**：从这3个样本学嵌入 → 不可靠，因为样本太少

**原型学习**：从所有时间戳中学习5个"原型"（典型的分布模式）：

|原型|含义|示例|
|---|---|---|
|**原型1**|早高峰模式（高用电量）|周一~周五 08:00-10:00|
|**原型2**|晚高峰模式（高用电量）|周一~周五 18:00-21:00|
|**原型3**|夜间低谷模式（低用电量）|每天 00:00-05:00|
|**原型4**|周末白天模式（中等用电量）|周六~周日 10:00-18:00|
|**原型5**|工作日白天模式（中等用电量）|周一~周五 10:00-17:00|

然后"周一早上8点"这个时间戳，就用它最相似的**原型1（早高峰模式）** 来表示，而不是从3个样本中硬学。

### 三、数学公式详解

#### 步骤1：计算相似度（公式5）

$$S_t = T_t P^\top$$

- $T_t \in \mathbb{R}^D$：时间戳 t 的原始嵌入（TiD + DiW 的和）
- $P = \{p_1, p_2, ..., p_N\} \in \mathbb{R}^{N \times D}$：原型库，包含 N 个可学习的原型向量
- $S_t \in \mathbb{R}^N$：时间戳 t 与每个原型的相似度得分（内积）

**例子**：假设有5个原型，计算得到：

|原型|相似度得分 $S_t$|
|---|---|
|原型1（早高峰）|**8.5** ✅ 最高|
|原型2（晚高峰）|2.1|
|原型3（夜间低谷）|-1.3|
|原型4（周末白天）|0.5|
|原型5（工作日白天）|**6.2**|

#### 步骤2：Top-k 选择 + Softmax（公式6）

$$w_{t,[1:k]} = \text{Softmax}(S_{t,[1:k]})$$

- 只保留最相似的 k 个原型（论文中 k=2）
- 对它们的得分做 Softmax，得到权重

**例子**（k=2）：

|选中的原型|原始得分|Softmax 后权重|
|---|---|---|
|原型1（早高峰）|8.5|**0.91**|
|原型5（工作日白天）|6.2|**0.09**|

其他原型（晚高峰、夜间低谷、周末白天）的权重直接设为 0。

#### 步骤3：加权聚合（公式7）

$$\tilde{T}_t = \sum_{j=1}^N W_{t,j} p_j$$

- $W_t \in \mathbb{R}^N$：稀疏权重向量（只有 top-k 非零）
- $\tilde{T}_t \in \mathbb{R}^D$：最终的时间戳表示

**例子**：

$$\tilde{T}_{Monday08:00} = 0.91 \times p_{早高峰} + 0.09 \times p_{工作日白天}$$

#### 完整流程示意图

```
时间戳: Monday 08:00
        │
        ▼
原始嵌入 T_t = Embedding("08:00") + Embedding("Monday")
        │
        ▼
与原型库 P 做内积 → S_t = T_t P^T
        │
        ▼
选择 top-2 最相似的原型
        │
        ├── 原型1（早高峰）: 得分 8.5 → Softmax → 权重 0.91
        └── 原型5（工作日）: 得分 6.2 → Softmax → 权重 0.09
        │
        ▼
加权聚合: T~_t = 0.91 × p₁ + 0.09 × p₅
        │
        ▼
送入 MLP 生成仿射参数: γ_t, β_t = MLP(T~_t)
```

### 四、为什么原型学习有效？

#### 1. 解决稀疏性问题

|方法|出现1次的时间戳|出现50次的时间戳|
|---|---|---|
|**直接学习嵌入**|❌ 不可靠|✅ 可靠|
|**原型学习**|✅ 借用相似原型的知识|✅ 同样受益|

即使某个时间戳只出现1次，只要它和"早高峰"原型相似，就能借用原型1的知识。

#### 2. 泛化到未见的时间戳

假设测试集中出现了**训练集没有的**时间戳组合（如某个节假日的特殊时段）：

|方法|未见的时间戳|
|---|---|
|**直接学习嵌入**|❌ 没有对应的嵌入，无法处理|
|**原型学习**|✅ 计算与已知原型的相似度，用最相似的原型表示|

#### 3. 可解释性

每个原型对应一种**典型的分布模式**：

|原型|对应的分布特征|
|---|---|
|早高峰原型|均值高、方差大|
|夜间低谷原型|均值低、方差小|
|周末原型|均值中等、模式特殊|

这让 APT 的决策过程变得可解释。

### 五、论文中的关键设计细节

#### 原型数量 N

论文根据数据频率选择原型数：

|数据集|采样频率|原型数 N|
|---|---|---|
|ECL|10分钟|**40**|
|ETTh1/ETTh2|1小时|**30**|
|Exchange|1天|**5**|

#### Top-k 选择

论文使用 k=2，原因：

> 论文原文：*"We set k=2 as a default, which provides sufficient sparsity while maintaining representational capacity."*

- k=1：太极端，只用一个原型，丢失信息
- k=2：稀疏但保留一定灵活性
- k=3+：太密集，失去原型学习的稀疏性优势

#### 同时编码历史和未来

$$\tilde{T}_t = \tilde{T}^{history}_t + \tilde{T}^{future}_t$$

- 历史上下文：输入序列的时间戳信息
- 未来上下文：预测目标的时间戳信息
- 两者相加，让 APT 同时知道"现在是什么分布"和"未来可能是什么分布"

### 六、消融实验验证

论文做了消融实验验证原型学习的必要性：

|变体|性能变化|原因|
|---|---|---|
|**W/O Prototype**（直接用原始时间戳嵌入）|**下降**|稀疏时间戳的嵌入不可靠|
|**W/O Top-k**（使用所有原型）|**下降**|引入不相关原型的噪声|
|**完整 APT**|**最佳** ✅|稀疏 + 原型 = 最佳效果|

### 一句话总结

> **原型学习就是先学习一组"典型的分布模式"（原型），然后每个时间戳不再用自己的原始嵌入，而是找到最相似的几个原型，用它们的加权组合来表示自己。这解决了时间戳组合稀疏的问题，让 APT 即使面对训练中没见过的时刻也能泛化——因为只要它和某个已知原型相似，就能借用那个原型的知识。**
