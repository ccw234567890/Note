### 一、Rollout 是什么？

**Rollout** 是强化学习中的术语，指**从初始状态开始，让策略（policy）与环境交互直到终止条件的一整轮执行过程**。

可以理解为一次"从头到尾的完整尝试"：

- 在游戏场景中：一局游戏 = 一次 rollout
- 在机器人场景中：机器人从起始位置开始执行任务直到成功或失败 = 一次 rollout

论文中说"每个任务最终策略在 **10 次不同初始状态的 rollout** 上评估成功率"，意思是：把机器人放在 10 种不同的起始位置/条件下，各跑一次完整任务，看成功几次。比如擦白板任务成功 9/10 次，成功率就是 0.9。

### 二、观测空间的处理流程详解

整个观测处理流水线如下：

```
原始观测 → 320×240 RGBD图像 → 分割降采样 → 50个点云点 → PointNet → 特征向量
                                                                      ↓
                                                        与7维本体感知向量拼接
                                                                      ↓
                                                        送入MLP网络 → 输出离散隐式动作
```

#### 2.1 320×240 RGBD 图像

- **RGBD** = RGB（彩色图像，3通道）+ **Depth**（深度图像，1通道）
- 320×240 是分辨率（宽×高）
- 所以原始输入是一个 **320×240×4** 的张量（R、G、B、D 四个通道）
- 这相当于一个低分辨率的摄像头画面，每个像素不仅有颜色信息，还有距离信息

#### 2.2 分割降采样至 50 个点云点

这一步做了两件事：

**分割（Segmentation）**：从 RGBD 图像中识别出属于"感兴趣物体"的像素（比如白板、桌子、垃圾等），把背景去掉。

**降采样（Downsampling）**：分割后可能还有成千上万个像素点，但为了计算效率，只随机采样 **50 个点**。

每个点包含的信息是 **(x, y, z)** 三维空间坐标（由深度信息反投影得到），所以最终得到 **50×3** 的点云矩阵。

> 为什么只保留 50 个点？因为机器人只需要知道物体的大致位置和形状，不需要高精度细节。50 个点已经足够，计算量却大幅降低。

#### 2.3 PointNet

**PointNet** 是 2017 年提出的经典点云处理神经网络架构，专门处理无序的点集。

核心思想：对每个点独立提取特征，然后用**对称函数（如 max pooling）** 聚合所有点的信息，得到一个全局特征向量。

数学上：

$$f(x_1, x_2, ..., x_n) = \text{MAX}\left( h(x_1), h(x_2), ..., h(x_n) \right)$$

其中 $h$ 是 MLP（多层感知机），MAX 是逐元素取最大值。

关键特性：

- **置换不变性**：无论点的顺序如何，输出相同（因为 max pooling 与顺序无关）
- 输入 50×3 → 输出一个固定维度的特征向量（比如 256 维）

#### 2.4 7 维本体感知向量

**本体感知（Proprioception）** 是机器人对自己身体状态的感知，就像人类闭着眼睛也能知道自己的手臂在什么位置。

这 7 维向量通常包含：

- 机器人各关节的角度或位置（比如头部相机俯仰角、左右臂的位姿等）
- 可能还包括末端执行器的速度、力传感器读数等

#### 2.5 拼接（Concatenation）

PointNet 输出的点云特征向量（假设 256 维）与 7 维本体感知向量**拼接**在一起，形成一个 **263 维**的联合特征向量：

$$\text{feature} = [\text{pointnet\_features}_{256}, \text{proprioception}_7]$$

这个联合向量包含了机器人对外部环境的理解（点云）和对自身状态的感知（本体）。

#### 2.6 MLP 网络 → 输出离散隐式动作

**MLP（多层感知机）** = 全连接神经网络，由若干层线性层 + 激活函数（ReLU）堆叠而成。

论文中下游任务策略的 MLP 尺寸为 **[256, 256]**，即两层隐藏层，每层 256 个神经元。

输入是 263 维联合特征向量，经过 MLP 处理后，输出**离散隐式动作**。

#### 2.7 离散隐式动作（Discrete Latent Action）

这是 SLAC 的核心概念：

- **隐式动作（Latent Action）**：不是直接控制机器人电机（低层动作），而是一个**抽象的高层指令**，比如"向左移动""伸手""旋转"等。解码器 πdec 负责把隐式动作翻译成具体的电机控制信号。
- **离散（Discrete）**：隐式动作空间是离散的，论文中维度为 **45**（可能分解为 5 个维度，每个维度 9 个离散值，组合成 45 种可能）。
- **时间扩展（Temporally Extended）**：一个隐式动作持续执行 **50 步**低层动作（steps_per_skill = 50），而不是每步都重新决策。这大大降低了决策频率，使探索更高效。

### 四、类比理解

想象你在玩一个**遥控机器人**的游戏：

|概念|类比|
|---|---|
|**RGBD 图像**|你通过摄像头看房间，知道物体颜色和距离|
|**50 个点云点**|你只扫了一眼，记住房间里 50 个关键物体的位置|
|**PointNet**|你的大脑快速处理这些位置信息，形成"房间布局"的概念|
|**7 维本体感知**|你闭眼也知道自己手臂在什么位置|
|**MLP 网络**|你的决策中枢，综合所有信息做判断|
|**隐式动作 z**|你决定"向左走 3 步"（高层指令），而不是"左腿肌肉收缩 5 度"|
|**解码器 πdec**|你的身体自动把"向左走 3 步"翻译成具体的肌肉运动序列|
|**Rollout**|你从起点开始完整走一遍任务，看能否成功|

SLAC 的创新在于：**阶段一先在仿真中学会"如何控制身体"（解码器），阶段二在真实世界中只需学习"如何做决策"（任务策略）**，大大降低了真实世界学习的难度和风险。

### 点云矩阵（Point Cloud Matrix）

**点云矩阵** 就是把三维空间中的一组点，用矩阵的形式表示出来。

#### 基本形式

假设有 N 个点，每个点有 (x, y, z) 三个坐标，那么点云矩阵就是一个 **N × 3** 的矩阵：

```
点云矩阵 = [x₁  y₁  z₁]     ← 第1个点
           [x₂  y₂  z₂]     ← 第2个点
           [x₃  y₃  z₃]     ← 第3个点
           ...
           [xₙ  yₙ  zₙ]     ← 第N个点
```

在 SLAC 论文中，分割降采样后得到 **50 个点云点**，所以点云矩阵的形状是 **50 × 3**：

```
点云矩阵 = [x₁  y₁  z₁]     ← 白板左上角的一个点
           [x₂  y₂  z₂]     ← 白板中间的一个点
           [x₃  y₃  z₃]     ← 白板右下角的一个点
           ...
           [x₅₀ y₅₀ z₅₀]   ← 第50个点
```

#### 每个点的含义

每个点 (x, y, z) 代表空间中一个位置的三维坐标：

|坐标|含义|示例值|
|---|---|---|
|**x**|左右方向（米）|0.35|
|**y**|前后方向（米）|0.82|
|**z**|上下方向（米）|0.15|

这些坐标是从 RGBD 图像的深度信息反算出来的——相机知道每个像素的颜色和距离，通过相机内参就能把 (像素行, 像素列, 深度值) 换算成 (x, y, z) 三维坐标。

#### 为什么叫"矩阵"？

因为它在数学上就是一个标准的矩阵（Matrix）——**行代表样本（点），列代表特征（坐标）**：

$$\text{PointCloud} = \begin{bmatrix} 
x_1 & y_1 & z_1 \\
x_2 & y_2 & z_2 \\
\vdots & \vdots & \vdots \\
x_{50} & y_{50} & z_{50}
\end{bmatrix}_{50 \times 3}$$

#### 直观类比

想象你在一个黑暗房间里用手电筒照一个物体：

1. **RGBD 图像** = 你用手电筒扫了一圈，看到物体的大致轮廓和远近
2. **分割** = 你只关注那个物体，忽略背景
3. **降采样到 50 个点** = 你闭眼在物体表面随机戳了 50 个点，记住每个点的位置
4. **点云矩阵** = 把这 50 个点的 (x, y, z) 坐标写在一张表格上，每行一个点

#### 为什么用矩阵形式？

因为神经网络（PointNet）的输入要求是**固定形状的张量**。把点云写成 50×3 的矩阵，就可以直接喂给神经网络处理。PointNet 会对每一行（每个点）独立提取特征，然后用 max pooling 聚合所有行的信息，最终输出一个固定维度的全局特征向量。

#### 与图像的区别

||图像（Image）|点云矩阵（Point Cloud Matrix）|
|---|---|---|
|**数据结构**|规则的网格（H×W×C）|无序的点集（N×3）|
|**排列方式**|像素有固定行列顺序|点的顺序可以任意打乱|
|**信息**|每个像素有颜色值|每个点有三维坐标|
|**优点**|纹理丰富|直接表达三维几何结构|

这就是为什么机器人任务中常用点云而非纯图像——机器人需要知道物体在**三维空间中的精确位置**才能去抓、推、擦，而点云矩阵正好提供了这种信息。
