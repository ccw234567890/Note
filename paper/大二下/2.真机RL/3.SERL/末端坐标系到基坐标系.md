### 末端坐标系 → 基坐标系：伴随映射

#### 问题场景

在SERL中，RL策略输出的6D twist动作 $\mathfrak{v}^{(i)}_t$ 是在**当前末端执行器坐标系** $\{b^{(i)}_t\}$ 中表达的。但机器人的底层控制软件期望接收的是**基坐标系** $\{s\}$ 中的动作。

> Mathematically, the 6 DoF twist action $\mathfrak{v}^{(i)}_t$ expressed in frame $\{b^{(i)}_t\}$ at timestep $t$. To interface with the robot's control software, which expects actions $\mathfrak{v}'^{(i)}_t$ expressed in the base frame $\{s\}$, we apply the Adjoint mapping.

(Luo 等, 2025)

---

#### 直观理解：坐标系视角的差异

想象你坐在一辆车里：

```
你在车里伸手（末端坐标系视角）：
  - "手向前伸"  → 这是相对于车头方向
  - "手向右伸"  → 这是相对于车身右侧

地面上的人看（基坐标系视角）：
  - 如果车头朝北，"手向前伸" = 向北移动
  - 如果车头朝东，"手向前伸" = 向东移动
  - 同一个动作，在不同坐标系下表达不同！
```

**伴随映射就是"翻译官"**：把"车里视角"的动作翻译成"地面视角"的动作。

---

#### 数学公式

伴随映射的公式为：

$$\mathfrak{v}'^{(i)}_t = [Ad^{(i)}_t] \mathfrak{v}^{(i)}_t$$

(Luo 等, 2025)

其中伴随矩阵 $[Ad^{(i)}_t]$ 定义为：

$$[Ad^{(i)}_t] = \begin{bmatrix} R_{b^{(i)}_t} & 0_{3\times3} \\ [p_{b^{(i)}_t}]_\times R_{b^{(i)}_t} & R_{b^{(i)}_t} \end{bmatrix}$$

(Luo 等, 2025)

---

#### 分步拆解

##### 1. 什么是6D Twist？

一个twist由两部分组成：

$$\mathfrak{v} = \begin{bmatrix} \omega \\ v \end{bmatrix}$$

|分量|名称|维度|含义|
|---|---|---|---|
|**$omega$**|角速度|$3times1$|绕x/y/z轴的旋转速度|
|**$v$**|线速度|$3times1$|沿x/y/z轴的移动速度|

##### 2. 伴随矩阵中的符号含义

|符号|含义|
|---|---|
|**$R_{b^{(i)}_t}$**|$3times3$ 旋转矩阵——描述末端坐标系相对于基坐标系的朝向|
|**$p_{b^{(i)}_t}$**|$3times1$ 平移向量——末端坐标系原点在基坐标系中的位置|
|**$[p]_times$**|反对称矩阵——将叉乘转化为矩阵乘法|
|**$0_{3times3}$**|$3times3$ 零矩阵|

##### 3. 反对称矩阵是什么？

对于向量 $p = [p_x, p_y, p_z]^T$，反对称矩阵定义为：

$$[p]_\times = \begin{bmatrix} 0 & -p_z & p_y \\ p_z & 0 & -p_x \\ -p_y & p_x & 0 \end{bmatrix}$$

它的作用是：$[p]_\times \cdot v = p \times v$（将叉乘转化为矩阵乘法）

---

#### 伴随映射的物理意义

将公式展开，看看每个分量如何变换：

```
角速度变换： ω' = R · ω
              ↑
    末端坐标系中的旋转 → 基坐标系中的旋转
    （纯旋转，与位置无关）

线速度变换： v' = [p]× · R · ω + R · v
              ↑              ↑
          平移-旋转耦合项    纯平移变换
```

##### 为什么线速度变换多了一项？

因为**坐标系原点不同**。当末端坐标系绕基坐标系中的某点旋转时，即使末端坐标系中"没有平移"，在基坐标系看来也会产生线速度。

**一个具体例子**：

```
基坐标系原点 ────────────── 末端坐标系原点
    (0,0,0)    距离=1m     (1,0,0)

末端坐标系中的动作：
  ω = [0, 0, 1]  (绕z轴旋转)
  v = [0, 0, 0]  (没有平移)

基坐标系中看到的动作：
  ω' = R·ω = [0, 0, 1]  (同样绕z轴旋转)
  v' = [p]×·R·ω + R·v
     = [p]×·ω           (因为R=I, v=0)
     = [1,0,0]× × [0,0,1]
     = [0, -1, 0]       (产生了线速度！)
```

**直观理解**：你站在旋转木马边缘（距离中心1m），即使你站在原地不动（末端坐标系中 $v=0$），旋转木马转动时，地面上的人看到你在**绕圈运动**（基坐标系中 $v \neq 0$）。这个"绕圈运动"就是耦合项 $[p]_\times R\omega$ 描述的。

---

#### 在SERL中的完整流程

```
┌────────────────────────────────────────────────────────┐
│  1. RL策略输出动作（末端坐标系视角）                      │
│     𝔳 = [ω_end, v_end]^T                               │
│     例如: "手向前伸5cm，顺时针转2度"                      │
│                                                        │
│  2. 获取当前末端位姿                                    │
│     R = 末端相对于基座的旋转矩阵                          │
│     p = 末端原点在基座中的位置                            │
│                                                        │
│  3. 构造伴随矩阵                                        │
│     [Ad] = [[R, 0], [[p]×R, R]]                        │
│                                                        │
│  4. 应用伴随映射                                        │
│     𝔳' = [Ad] · 𝔳                                      │
│     例如: "向北移动3cm，向东移动4cm，绕垂直轴转2度"        │
│                                                        │
│  5. 将𝔳'发送给底层控制软件执行（基坐标系视角）             │
└────────────────────────────────────────────────────────┘
```

---

#### 为什么这样做？

论文指出，使用相对坐标系 + 伴随映射带来了关键优势：

> As a result, the policy can succeed even if the object moves or, as in some of our experiments, is perturbed in the middle of the episode.

(Luo 等, 2025)

|如果不做伴随映射|如果做伴随映射|
|---|---|
|策略学习"绝对位置"|策略学习"相对运动"|
|物体移动后策略失效|物体移动后策略自动适应|
|需要大量数据覆盖所有位置|少量数据即可泛化|
|训练慢|训练快（20分钟收敛）|

---

#### 总结

```
末端坐标系 → 基坐标系的伴随映射

本质：坐标系变换的"翻译官"
输入：末端视角的6D twist动作
输出：基座视角的6D twist动作
公式：𝔳' = [Ad] · 𝔳
矩阵：[Ad] = [[R, 0], [[p]×R, R]]

物理意义：
  ω' = R·ω           ← 旋转直接旋转
  v' = [p]×·R·ω + R·v ← 平移 = 耦合项 + 纯平移

效果：策略在末端视角思考，机器人在基座视角执行
```
