### 一、奖励函数（Reward Functions）——三种方式

SERL提供了三种奖励函数设计方式，按自动化程度从高到低排列：

#### 方式1：二分类器奖励（Binary Classifier）

> The reward functions can be specified either with a binary classifier

**原理**：训练一个二分类器，判断当前状态是否"成功"。奖励就是分类器输出"成功"概率的对数：

$$r(s) = \log p(e|s)$$

其中 $p(e|s)$ 是分类器认为状态 $s$ 下任务成功 $e$ 的概率。

**适用场景**：当无法手工定义奖励函数时（例如线缆布线任务——很难用数学公式判断"线缆是否卡入夹子"）。

**训练数据**：需要少量成功状态的演示（正样本）和随机状态的快照（负样本）。

#### 方式2：VICE（Variational Inverse Control with Events）

> or VICE (Fu et al., 2018), which provides a method to update the classifier during RL training with additional negatives from the policy

**核心创新**：VICE在RL训练过程中**动态更新分类器**。

**为什么需要VICE？** 普通二分类器有一个严重问题——**分布偏移（distribution shift）**。初始训练时，分类器只在演示数据上训练，但RL策略在探索过程中会访问到分类器从未见过的状态。如果策略偶然进入一个分类器误判为"成功"的状态，就会获得虚假的高奖励，导致策略"欺骗"分类器。

**VICE的解决方案**：每轮RL训练后，将策略访问过的所有状态作为**新的负样本**加入分类器训练集，重新训练分类器。这形成了一个类似GAN的对抗过程：

```
策略（生成器）← 试图找到分类器误判为成功的状态
分类器（判别器）← 不断学习区分策略的新状态和真实成功状态
```

#### 方式3：手工指定奖励（Hand-specified）

> The reward function can also be specified by hand in cases where the robot state is sufficient to evaluate success (e.g., as in our PCB board assembly task)

**适用场景**：当任务成功条件可以用机器人状态直接判断时。

**PCB插装任务的例子**：判断连接器是否插入成功，只需要检查末端执行器的**位置和力传感器读数**：

```
if (z_position < 阈值) and (force_z > 阈值):
    reward = 1  # 成功
else:
    reward = 0  # 未成功
```

这种情况下不需要视觉分类器，因为状态空间足够简单。

### 二、自动重置（Automatic Resets）——前向-后向架构

> The resets can be provided via a forward-backward architecture (Sharma et al., 2021), where the algorithm simultaneously trains two policies: a forward policy that performs the task, and a backward policy that resets the environment back to the initial state

#### 为什么需要自动重置？

在真实机器人上做RL有一个**致命问题**：每个训练回合结束后，需要有人手动把机器人重置到初始状态。如果训练几万步，就需要人一直守在旁边手动重置，这完全违背了"自动化"的初衷。

#### 前向-后向架构的解决方案

同时训练**两个策略**：

```
前向策略（Forward Policy）：执行任务本身
    └─ 例如：把物体从箱子A搬到箱子B

后向策略（Backward Policy）：把环境重置回初始状态
    └─ 例如：把物体从箱子B搬回箱子A
```

#### 训练流程

```
回合开始 → 前向策略执行任务 → 任务完成或超时
                              ↓
                   后向策略执行重置 → 回到初始状态
                              ↓
                   前向策略开始下一回合
```

**关键优势**：整个过程完全自动化，不需要人类干预。

#### 在物体搬运任务中的具体例子

```
前向策略：把红色方块从左侧箱子抓到右侧箱子
后向策略：把红色方块从右侧箱子抓回左侧箱子

训练循环：
  第1回合：前向策略执行（成功搬运）→ 后向策略执行（搬回）
  第2回合：前向策略执行（又成功）→ 后向策略执行（搬回）
  ...无限循环，无需人类介入
```

### 三、机器人系统适配层

> On the robot system side, we also provide a universal adapter for interfacing our method to arbitrary robots

**含义**：SERL提供了一个通用适配器，可以将RL算法输出的控制指令转换为任意机器人平台能理解的格式。

**为什么需要这个？** 不同机器人厂商的接口完全不同：

SERL的适配器抽象了这些差异，让RL代码与机器人硬件解耦。

### 四、阻抗控制器（Impedance Controller）

> as well as an impedance controller that is particularly well-suited for contact-rich manipulation tasks

#### 什么是阻抗控制器？

阻抗控制器模拟一个**弹簧-阻尼系统**：

$$F = k_p \cdot e + k_d \cdot \dot{e}$$

其中：

- $e = p - p_{ref}$：当前位置与目标位置的误差
- $k_p$：刚度（弹簧常数）
- $k_d$：阻尼系数

#### 为什么适合接触丰富的操作任务？

**接触丰富的任务**（如PCB插装、拧螺丝、装配）的特点是：机器人需要与环境发生物理接触，而接触力很难精确控制。

**阻抗控制器的优势**：

|特性|说明|
|---|---|
|**柔顺性**|遇到阻力时自动"让开"，而不是硬顶|
|**安全性**|限制最大接触力，保护机器人和工件|
|**无需精确建模**|不需要知道接触点的精确物理参数|

**PCB插装的具体例子**：

```
RL策略输出：向下移动2mm（期望位置）
阻抗控制器执行：
  - 如果连接器对准了插孔 → 轻松插入，误差小，力小
  - 如果连接器没对准 → 遇到阻力，误差增大
    → 控制器自动降低力，防止损坏PCB
    → 同时RL策略从力反馈中学习到"需要调整角度"
```

### Q 到底是什么？

#### 数学定义

在 SAC 中，Q 函数（Soft Q 函数）定义为：

> $$Q^\pi(s_t,a_t) = r(s_t,a_t) + \mathbb{E}_{(s_{t+1},\dots)\sim\rho_\pi}\left[\sum_{l=1}^\infty \gamma^l \big(r(s_{t+l},a_{t+l}) + \alpha\mathcal{H}(\pi(\cdot|s_{t+l}))\big)\right]$$

(Haarnoja 等, 2019)

翻译成大白话：**在状态 $s_t$ 执行动作 $a_t$ 后，按照策略 $\pi$ 继续行动，所能获得的"总收益"的期望值。**

这个"总收益"包含两部分：

1. **累积奖励** $\sum \gamma^l r$ — 未来每一步获得的即时奖励（折扣求和）
2. **累积熵** $\sum \gamma^l \alpha\mathcal{H}$ — 未来每一步策略的随机性奖励（折扣求和）

#### 和传统 Q 函数的区别

||传统 Q 函数（DQN/DDPG）|SAC 的 Soft Q 函数|
|---|---|---|
|定义|$Q = mathbb{E}[sum gamma^l r]$|$Q = mathbb{E}[sum gamma^l (r + alphamathcal{H})]$|
|考虑了什么|只有奖励|奖励 + 策略的熵|
|意义|"这个动作能赚多少分"|"这个动作能赚多少分 + 未来还有多少探索空间"|

#### 直观理解

想象你在玩一个游戏：

- **传统 Q 函数**告诉你：在状态 $s$ 做动作 $a$，未来能拿到的总分数是 100 分
- **SAC 的 Soft Q 函数**告诉你：在状态 $s$ 做动作 $a$，未来能拿到的总分数是 100 分，**而且未来策略还有足够的随机性去探索更好的路径**，这额外值 5 分

#### SAC 中 Q 函数的具体形式

在实际算法中，Q 函数是一个**神经网络** $Q_\theta(s,a)$，它接收状态和动作作为输入，输出一个标量值。

论文中 Q 函数的更新公式（公式 5）：

> $$J_Q(\theta) = \mathbb{E}_{(s_t,a_t)\sim\mathcal{D}}\left[\frac{1}{2}\left(Q_\theta(s_t,a_t) - \hat{Q}(s_t,a_t)\right)^2\right]$$

(Haarnoja 等, 2019)

其中目标值 $\hat{Q}$ 为：

> $$\hat{Q}(s_t,a_t) = r(s_t,a_t) + \gamma\mathbb{E}_{s_{t+1}\sim p}[V_{\bar{\theta}}(s_{t+1})]$$

(Haarnoja 等, 2019)

而 $V_{\bar{\theta}}(s_{t+1})$ 是 Soft 状态值函数：

> $$V_{\bar{\theta}}(s_{t+1}) = \mathbb{E}_{a_{t+1}\sim\pi_\phi}[Q_{\bar{\theta}}(s_{t+1},a_{t+1}) - \alpha\log\pi_\phi(a_{t+1}|s_{t+1})]$$

(Haarnoja 等, 2019)

#### Q 在 SAC 中的角色

```
┌─────────────────────┐
                    │    Actor (策略 π)    │
                    │  输出动作 a_t        │
                    └────────┬────────────┘
                             │
                             ▼
                    ┌─────────────────────┐
         ┌─────────│  Critic (Q 函数 Q_θ) │←──────── 环境反馈 r_t
         │         │  评估动作好坏        │
         │         └────────┬────────────┘
         │                  │
         │    Q 值反馈给 Actor 更新策略
         ▼
    ┌─────────────────────────────┐
    │ Actor 更新目标：             │
    │ min E[αlogπ(a|s) - Q(s,a)]  │
    │ 即：让策略倾向 Q 值高的动作  │
    └─────────────────────────────┘
```

**Q 函数是 Actor 的"老师"**：

- Actor 输出动作
- Q 函数评价这个动作好不好
- Actor 根据 Q 函数的评价来改进自己

#### 为什么 SAC 用两个 Q 函数？

SAC 实际维护两个 Q 网络 $Q_{\theta_1}$ 和 $Q_{\theta_2}$，取**最小值**作为最终 Q 值：

$$Q(s_t,a_t) = \min(Q_{\theta_1}(s_t,a_t), Q_{\theta_2}(s_t,a_t))$$

这是为了防止 **Q 值过高估计**——神经网络有时会"过于乐观"，两个网络互相制约，取最小值更保守、更稳定。

#### 总结

**Q 函数就是 SAC 中 Critic 的角色**，它回答一个问题：

> **"在当前状态做这个动作，未来到底好不好？"**

Actor 根据 Q 函数的答案来调整自己的策略，Q 函数则根据环境的反馈（奖励）不断修正自己的判断。两者相互促进，共同进步。
