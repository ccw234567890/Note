### 累积回报（Cumulative Return）

**累积回报**是指智能体从某个时间步开始，到任务结束为止，所获得的**所有即时奖励的总和**。

对于一个长度为 T 的轨迹（trajectory）：

> τ = (s₀, a₀, r₀, s₁, a₁, r₁, ..., s_T, a_T, r_T)

累积回报定义为：

> G(τ) = r₀ + r₁ + r₂ + ... + r_T = ∑_{t=0}^{T} r(s_t, a_t)

**直观理解**：就像你玩游戏时，把每一关获得的分数全部加起来得到总分。

#### 问题：无限时间步的情况

很多真实世界任务（包括SERL中的机器人操作任务）是**没有明确结束时间**的——理论上可以无限运行下去。如果任务无限长（T → ∞），累积回报会变成：

> G(τ) = r₀ + r₁ + r₂ + ... = ∑_{t=0}^{∞} r(s_t, a_t)

这里出现了一个**数学问题**：如果每个时间步的奖励都是正数，这个无穷级数会**发散到无穷大**，无法比较不同策略的好坏。

---

### 折扣累积回报（Discounted Cumulative Return）

为了解决上述问题，引入了**折扣因子 γ ∈ [0, 1)**：

> G(τ) = r₀ + γr₁ + γ²r₂ + γ³r₃ + ... = ∑_{t=0}^{∞} γ^t r(s_t, a_t)

(Luo 等, 2025)

#### 数学原理：几何级数

这是一个**几何级数**（geometric series）。如果奖励有界（|r_t| ≤ R_max），则：

> ∑_{t=0}^{∞} γ^t r_t ≤ R_max · ∑_{t=0}^{∞} γ^t = R_max · 1/(1-γ)

由于 γ < 1，这个级数**收敛到有限值**，解决了发散问题。

#### 直观理解

折扣因子 γ 的含义是**"未来的奖励值多少钱"**：

|γ 的值|含义|效果|
|---|---|---|
|γ → 0|**短视**|只看重眼前的奖励，不管未来|
|γ → 1|**有远见**|几乎同等看重所有时间步的奖励|
|γ = 0.99（SERL使用）|**适度远视**|100步后的奖励价值约为现在的 0.99¹⁰⁰ ≈ 36.6%|

#### 为什么需要折扣？

1. **数学收敛性**：确保无限时间步的累积回报是有限值
2. **经济直觉**：今天的1块钱比明天的1块钱更值钱——你可以把今天的钱存银行赚利息
3. **不确定性**：越远的未来越不确定，应该给予更低的权重
4. **促进快速完成任务**：折扣因子会"惩罚"拖沓——如果完成任务有正奖励，智能体会倾向于尽快完成，因为拖得越久，奖励被折扣得越多

#### 在SERL中的实际应用

论文中的优化目标正是**最大化折扣累积回报的期望**：

> E[∑_{t=0}^{∞} γ^t r(s_t, a_t)]

(Luo 等, 2025)

这个期望是对三个随机来源求平均：

- **初始状态随机性**：s₀ ∼ ρ(·)
- **转移随机性**：s_{t+1} ∼ (·|s_t, a_t)
- **策略随机性**：a_t ∼ π(·|s_t)

---

### 一个具体例子

假设一个机器人插装任务，γ = 0.9：

|时间步|奖励|折扣因子 γ^t|折扣后奖励|
|---|---|---|---|
|t=0|0|0.9⁰ = 1.0|0|
|t=1|0|0.9¹ = 0.9|0|
|t=2|0|0.9² = 0.81|0|
|t=3|1（成功插入）|0.9³ = 0.729|0.729|

**累积回报**：0 + 0 + 0 + 1 = 1

**折扣累积回报**：0 + 0 + 0 + 0.729 = 0.729

如果机器人花了10步才成功（γ¹⁰ = 0.349），折扣后奖励只有0.349——这就是折扣因子**鼓励更快完成任务**的体现。
