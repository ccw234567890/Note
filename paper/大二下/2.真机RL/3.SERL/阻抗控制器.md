### 阻抗控制器（Impedance Controller）详解

#### 一、背景：为什么需要阻抗控制器？

在SERL中，RL策略输出的是**期望末端位姿**（6D增量位姿），而不是直接输出关节力矩。因此需要一个**底层控制器**来将RL的高层指令转化为实际的关节力矩，并安全地执行。

SERL采用**两层控制层级结构**：

```
┌─────────────────────────────────────────────────────┐
│  高层：RL策略 π(a|s)          @ 10 Hz               │
│  输出：期望末端位姿 p_ref                              │
│         ↓                                            │
│  每个RL时间步 = 100个控制时间步                          │
│         ↓                                            │
│  低层：阻抗控制器                @ 1 KHz              │
│  输出：关节力矩 τ                                      │
│         ↓                                            │
│  机器人执行                                            │
└─────────────────────────────────────────────────────┘
```

> a high-level RL controller π(a|s) sends control targets at 10HZ for the low-level impedance controller to track at 1K HZ, so one timestep from RL will block 100 timesteps of the low-level controller to execute.

(Luo 等, 2025)

---

#### 二、阻抗控制的核心公式

> A typical impedance control objective for this controller is

$$F = k_p \cdot e + k_d \cdot \dot{e} + F_{ff} + F_{cor}$$

(Luo 等, 2025)

其中：

- **$e = p - p_{ref}$**：当前末端位姿 $p$ 与目标位姿 $p_{ref}$ 之间的**误差**
- **$k_p$**：**刚度系数（stiffness）**，类似弹簧常数
- **$k_d$**：**阻尼系数（damping）**，类似减震器
- **$\dot{e}$**：误差的变化率（速度误差）
- **$F_{ff}$**：**前馈力（feed-forward force）**
- **$F_{cor}$**：**科里奥利力（Coriolis force）**，补偿机器人动力学中的惯性耦合

这个控制力 $F$ 随后通过**雅可比矩阵转置**转换为关节力矩：

$$\tau = J^T \cdot F + \text{nullspace torques}$$

---

#### 三、弹簧-阻尼系统的物理直觉

阻抗控制器的本质是一个**弹簧-阻尼系统（spring-damper system）**：

```
弹簧 (k_p)
    ┌───────╱╱╱╱╱╱╱╱╱╱╱╱╱╱╱───────┐
    │      ╱                      ╱      │
    │ 机器人 ◁═══════ 阻尼 (k_d) ═══════ ▷ 目标位姿 p_ref
    │  p  ╲                      ╲      │
    └───────╲╲╲╲╲╲╲╲╲╲╲╲╲╲╲───────┘
    
    弹簧力 = k_p · e    (将机器人拉向目标)
    阻尼力 = k_d · ė    (阻止震荡)
```

**直观理解**：

- **弹簧**：误差越大，拉力越大，把机器人拉向目标位置
- **阻尼**：运动越快，阻力越大，防止机器人"冲过头"产生震荡
- 两者结合 = 一个**柔顺的虚拟弹簧**，既精确跟踪目标，又不会硬碰硬

> It acts like a spring-damper system around the equilibrium set by $p_{ref}$ with the stiffness coefficient being $k_p$ and the damping coefficient being $k_d$.

(Luo 等, 2025)

---

#### 四、核心问题：接触时的力约束

当机器人与环境接触时，如果 $p_{ref}$ 远离当前位姿 $p$，弹簧-阻尼系统会产生**巨大的力**：

> this system will yield large forces if $p_{ref}$ is far away from the current pose, which can lead to a hard collision or damage when the arm is in contact with something.

(Luo 等, 2025)

例如PCB插装任务中，如果RL策略命令末端执行器"向下移动1cm"，但实际已经接触到了PCB板表面，阻抗控制器会试图强行执行这个1cm的位移，产生巨大的接触力——可能损坏PCB板或机器人。

---

#### 五、SERL的解决方案：参考限幅（Reference Limiting）

SERL的核心创新在于**不在RL层裁剪动作，而是在实时控制层进行参考限幅**。

##### 5.1 限幅公式

对误差 $e$ 进行限幅：

$$|e| \leq \Delta$$

从而产生的力被约束为：

$$F_{\text{max}} = k_p \cdot |\Delta| + 2k_d \cdot |\Delta| \cdot f$$

(Luo 等, 2025)

其中 $f$ 是控制频率（1 KHz）。

##### 5.2 为什么不在RL层直接裁剪？

> One might wonder if we should directly clip the action output by the RL policy. This might seem reasonable, but can be impractical in some scenarios.

(Luo 等, 2025)

**原因**：PCB插装需要**微米级**的接触力精度（$\Delta$ 在微米量级）。如果RL策略只能输出微米级的增量，那么从起始位置移动到插孔位置（可能相距10-20cm）需要**数十万步**，训练将极其缓慢。

##### 5.3 关键洞察：自由空间 vs 接触空间

SERL利用**时间尺度分离**来解决这个矛盾：

> It will not block the free space movement of the RL policy as long as $M \cdot |\Delta| \geq |a|_{\text{max}}$, where $M$ is the number of control time-steps inside a block.

(Luo 等, 2025)

其中：

- **$M$**：每个RL时间步内的控制步数（$M = 100$）
- **$|\Delta|$**：每步允许的最大位移（微米级）
- **$|a|_{\text{max}}$**：RL策略输出的最大位移

**在自由空间中**：

- RL输出一个大的位移 $a$（比如1cm）
- 100个控制步 × 微米级步长 = 总位移可达 $M \cdot |\Delta|$（比如100μm × 100 = 1cm）
- 限幅器**不阻挡**，机器人快速移动

**在接触时**：

- 机器人已经接触表面，实际位移 $p$ 无法继续增加
- 误差 $e = p - p_{ref}$ 被限幅到 $\Delta$
- 产生的力被严格约束在安全范围内

##### 5.4 实验验证

论文通过Figure 3展示了两种场景下的控制器行为：

```
左图：接触场景（接触硬表面）
  橙色（RL命令）: 持续向下移动
  红色（平滑后）: 平滑跟踪
  蓝色（限幅后）: 被限幅，不继续下压 ← 保护接触力
  绿色（实际位置）: 停在接触表面

右图：自由空间场景（快速移动）
  橙色（RL命令）: 快速向下移动
  红色（平滑后）: 平滑跟踪
  蓝色（限幅后）: 不被阻挡，跟随RL命令 ← 允许快速移动
  绿色（实际位置）: 快速到达目标
```

> the controller indeed clamps the reference whenever in contact, while permitting fast movement in free space.

(Luo 等, 2025)

#### 六、为什么不用力/力矩传感器？

论文讨论了使用外部力/力矩传感器的替代方案，并指出其缺点：

> (1) force/torque sensors can have significant noise, and obtaining the right hardware and calibration can be difficult; (2) even if we get such a threshold value, it's nontrivial to design robot motions to accommodate policy learning as well as obeying the force constraint.

(Luo 等, 2025)

**SERL的方案优势**：

- **无需额外硬件**：不需要力/力矩传感器
- **无需标定**：没有传感器噪声问题
- **简单有效**：纯软件层面的限幅
- **通用性**：可在任何力矩控制机器人上实现
