### 前向策略（Forward Policy）与后向策略（Backward Policy）

#### 背景：为什么需要前向-后向策略？

在真实世界机器人RL中，一个长期存在的难题是**环境重置（environment reset）**。对于PCB插装和线缆布线这类任务，机器人可以在每个episode结束后自动回到起始位置（因为物体被刚性夹持在夹爪中）。但对于**自由漂浮物体（free-floating object）** 的搬运任务，情况完全不同：

> The intricacies of reward inference and reset-free training become especially pronounced in the manipulation of such free-floating objects.

(Luo 等, 2025)

想象一下：机器人从左边箱子抓起一个物体放到右边箱子。episode结束后，物体已经在右边箱子里了。如果下一个episode需要再次从左边开始，**必须有人手动把物体放回左边**——这就是"重置问题"。

SERL的解决方案是：**不重置环境，而是训练两个互为逆向的策略**。

---

#### 前向策略 vs 后向策略

|维度|前向策略|后向策略|
|---|---|---|
|**任务**|从左边箱子抓取物体放到右边|从右边箱子抓取物体放回左边|
|**本质**|完成目标任务|**撤销**前向策略的操作|
|**训练数据**|20个Space Mouse演示|20个Space Mouse演示|
|**训练时间**|< 1小时|< 1小时|
|**奖励函数**|二分类器（物体在右边=成功）|二分类器（物体在左边=成功）|

论文原文：

> We define the **forward task** as picking up the object from the bin on the right side and placing it on the left, while the **backward task** moves the object back to the starting bin, **undoing the forward task**.

(Luo 等, 2025)


---

#### 为什么这能解决"重置"问题？

传统方法需要**手动重置**：

```
episode 1: 机器人从左边抓物体 → 放到右边
              ↓
         人类手动把物体放回左边
              ↓
episode 2: 机器人从左边抓物体 → 放到右边
              ↓
         人类手动把物体放回左边
              ↓
         ... (每次都需要人工干预)
```

SERL的前向-后向方法实现**自动重置**：

```
episode 1: 前向策略执行 → 物体到右边
              ↓
episode 2: 后向策略执行 → 物体回左边  ← 自动重置！
              ↓
episode 3: 前向策略执行 → 物体到右边
              ↓
episode 4: 后向策略执行 → 物体回左边  ← 自动重置！
              ↓
         ... (无限循环，无需人工干预)
```

---

#### 两个策略如何协同训练？

SERL的物体搬运任务实际上训练了**两个独立的RL策略**，每个策略都使用RLPD算法，各自有独立的回放缓冲区：

```
┌────────────────────────────────────────────────────────┐
│                    训练流程                              │
│                                                        │
│  1. 前向策略训练:                                       │
│     - 输入: 腕部相机 + 侧方相机 + 本体感知                │
│     - 输出: 6D末端增量位姿                               │
│     - 奖励: 二分类器判断"物体在右边箱子"                   │
│     - 20个前向演示初始化缓冲区                            │
│                                                        │
│  2. 后向策略训练:                                       │
│     - 输入: 腕部相机 + 侧方相机 + 本体感知                │
│     - 输出: 6D末端增量位姿                               │
│     - 奖励: 二分类器判断"物体在左边箱子"                   │
│     - 20个后向演示初始化缓冲区                            │
│                                                        │
│  3. 交替执行:                                           │
│     前向策略跑一个episode → 后向策略跑一个episode → 循环  │
│                                                        │
│  4. 效果: 无需人工重置，训练可以持续进行直到收敛            │
│                                                        │
└────────────────────────────────────────────────────────┘
```

#### 实验结果

> The free-floating object relocation task learns two policies (forward and backward), and total time amounts to **less than an hour per policy**.

(Luo 等, 2025)

---

#### 前向-后向策略 vs 其他自动重置方法

|方法|原理|优点|缺点|
|---|---|---|---|
|**SERL前向-后向**|训练两个互为逆向的策略|无需任何人工干预，完全自主|需要训练两个策略，总时间翻倍|
|**手工重置**|人类手动放回物体|简单直接|需要人在环，无法长时间自主训练|
|**多任务学习** (Gupta 2021)|训练一个策略同时学会正向和逆向|共享参数，更高效|训练更复杂，可能互相干扰|
|**Leave No Trace** (Eysenbach 2018)|学习一个"重置策略"|通用框架|重置策略本身也需要训练|

SERL选择**训练两个独立策略**的方式，虽然总训练时间翻倍，但每个策略的任务更简单、更专注，最终实现了**100%的成功率**。

---

#### 总结

前向策略和后向策略是SERL解决**自由漂浮物体操作中环境重置问题**的核心方案：

1. **前向策略**：完成目标任务（左边→右边）
2. **后向策略**：撤销前向操作，自动重置环境（右边→左边）
3. **交替执行**：两个策略轮流运行，形成无限训练循环
4. **无需人工干预**：训练可以持续进行直到策略收敛

> Our framework does have a number of limitations... The full range of reward specifications and **reset-free learning challenges** still constitute an open problem in robotic RL research. Our classifier-based rewards and **forward-backward controller** might not be appropriate in every setting.

(Luo 等, 2025)

论文也坦诚指出，前向-后向方法并非万能，但在自由漂浮物体操作这一特定场景下，它提供了一个优雅且有效的解决方案。
