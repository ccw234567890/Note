论文中虽然没有给出"轨迹"的单独定义，但从 DAGGER 算法的描述中可以清楚地看出它的含义：

> Sample **T-step trajectories** using π_i. Get dataset D_i = {(s, π*(s))} of **visited states** by π_i and actions given by expert.

(Ross 等, 2010)

**轨迹（trajectory）** 在本文中的含义是：**从时间步 1 到时间步 T 的完整状态-动作序列**。

具体来说：

```
轨迹 = (s₁, a₁) → (s₂, a₂) → (s₃, a₃) → ... → (s_T, a_T)
```

其中：

- s_t 是时间步 t 的**状态（state）**
- a_t = π(s_t) 是学习器在该状态下选择的**动作（action）**
- 箭头表示状态转移：s_{t+1} 由 (s_t, a_t) 和环境共同决定

**所以你的理解基本正确，但更精确地说：**

轨迹不仅仅是"每次做完动作后的 state"，而是 **state 和 action 交替出现的完整序列**。每个时间步的流程是：

```
当前状态 s_t → 做出动作 a_t → 转移到下一个状态 s_{t+1} → 做出动作 a_{t+1} → ...
```

在 DAGGER 中，每次迭代都会用当前策略 π_i 采样 **T 步长的轨迹**，然后从这些轨迹中提取所有访问过的状态 s，并让专家为这些状态标注正确的动作 π*(s)，从而构建新的训练集 D_i。
