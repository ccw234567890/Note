
# $π_0$: A Vision-Language-Action Flow Model for General Robot Control

## 动作分块架构（Action Chunking Architecture）详解

### 一、概念定义

**动作分块（Action Chunking）** 是指模型在**单个推理步骤中一次性预测未来 $H$ 个时间步的连续动作序列**，而非逐个自回归地预测每个动作。

> We use an action chunking architecture [57] with flow matching (a variant of diffusion) to represent complex continuous action distributions. This enables our model to control robots at frequencies of up to 50 Hz for dexterous tasks such as laundry folding.

(Black et al., 2026)

在 π₀ 中，动作块定义为：

$$A_t = [a_t, a_{t+1}, \dots, a_{t+H-1}]$$

其中 $H=50$ 是动作分块长度（action horizon），每个 $a_{t'} \in \mathbb{R}^d$ 是一个 $d$ 维的连续动作向量（如关节角度、末端执行器位姿等）。

### 二、工作原理

#### 2.1 输入序列结构

π₀ 的 Transformer 输入序列由三部分组成：

```
输入序列 = [图像令牌 | 语言令牌 | 状态令牌 | 动作令牌]
            ↑ 观测令牌 (ot)  ↑       ↑ 噪声动作块 (A_t^τ)
```

> The standard PaliGemma architecture takes in a sequence of images $[I_{t1}, \dots, I_{tn}]$ followed by a language prompt $l_t$. We add an input $q_t$ for the robot's proprioceptive state, which is mapped to the transformer embedding dimension using a linear projection. The final set of input tokens correspond to the noisy action chunk $A_t^\tau = [a_t^\tau, \dots, a_{t+H-1}^\tau]$, with the number of tokens equal to the action horizon ($H=50$ for our tasks).

(Black et al., 2026)

#### 2.2 分块因果注意力掩码（Chunked Causal Attention Mask）

π₀ 使用一种**分块因果注意力掩码**来控制令牌之间的信息流动。整个序列被划分为三个块：

```
块 1: 图像令牌 [I_t1, ..., I_tn]  → 块内全连接（双向注意力）
块 2: 语言令牌 [l_t] + 状态令牌 [q_t]  → 块内因果（从左到右）
块 3: 动作令牌 [a_t^τ, ..., a_{t+H-1}^τ]  → 块内全连接（双向注意力）
```

块间规则：
- **块 1 → 块 2 → 块 3**：前面的块可以"看到"后面的块？不——这是**因果掩码**，后面的令牌可以看到前面所有令牌，但前面的令牌看不到后面的令牌。
- 具体来说，**动作令牌（块 3）可以看到所有观测令牌（块 1+2）**，但观测令牌看不到动作令牌。
- 动作令牌之间**互相可见**（全连接），因为它们是并行生成的。

这种设计确保了：
1. 动作生成可以**充分关注**所有观测信息（图像、语言指令、机器人状态）
2. 动作块内的 $H$ 个动作可以**互相通信**，学习动作之间的时序相关性
3. 观测编码不受动作令牌的干扰

#### 2.3 共享前缀 KV Cache

推理时，观测令牌（图像 + 语言 + 状态）只需编码**一次**，然后其 Key/Value 被缓存，供后续 10 步流匹配迭代复用：

> Each time we predict a new action chunk $A_t$, we must encode each of the images $I_{t1}, \dots, I_{tn}$, run a forward pass on the tokens corresponding to $o_t$, and then run 10 steps of flow matching, where each step requires running a forward pass on the tokens corresponding to $A_t^\tau$ (the keys and values corresponding to $o_t$ are cached).

(Black et al., 2026)

这意味着：
- **观测前向传播**：1 次（32 ms）
- **动作前向传播**：10 次（共 27 ms）
- **总推理时间**：约 73 ms（板载）或 86 ms（离板）

#### 2.4 开环执行（Open-Loop Execution）

π₀ 采用**开环执行**策略，即生成的动作块被逐个执行，不进行动作聚合：

> We tried temporal ensembling [57] early on and found that it hurt policy performance, so we opted not to aggregate actions and instead execute action chunks open-loop.

(Black et al., 2026)

执行频率取决于机器人类型：
- **20 Hz 机器人**（UR5e, Franka）：每 0.8 秒推理一次（执行 16 个动作后）
- **50 Hz 机器人**（其他所有）：每 0.5 秒推理一次（执行 25 个动作后）

### 三、数学原理

#### 3.1 动作令牌的嵌入计算

每个噪声动作 $a_{t'}^\tau$ 在输入 Transformer 之前，需要被映射到嵌入空间。这个映射同时融合了流匹配时间步 $\tau$ 的信息：

$$e_{t'} = W_3 \cdot \text{swish}\left(W_2 \cdot \text{concat}\left(W_1 \cdot a_{t'}^\tau,\ \phi(\tau)\right)\right)$$

其中：
- $a_{t'}^\tau \in \mathbb{R}^d$：第 $t'$ 个时间步的噪声动作向量
- $\phi: \mathbb{R} \to \mathbb{R}^w$：正弦位置编码函数，将标量 $\tau$ 编码为 $w$ 维向量
- $W_1 \in \mathbb{R}^{w \times d}$：将动作向量投影到嵌入维度 $w$
- $\text{concat}(W_1 \cdot a_{t'}^\tau,\ \phi(\tau)) \in \mathbb{R}^{2w}$：拼接动作嵌入和时间步编码
- $W_2 \in \mathbb{R}^{w \times 2w}$：将拼接向量压缩回 $w$ 维
- $\text{swish}$：Swish 激活函数，$\text{swish}(x) = x \cdot \sigma(x)$，其中 $\sigma$ 是 Sigmoid 函数
- $W_3 \in \mathbb{R}^{w \times w}$：最终线性投影
- $e_{t'} \in \mathbb{R}^w$：最终输入 Transformer 的动作令牌嵌入

#### 3.2 输出解码

vθ​ 是什么？

vθ是 π₀ 模型中可学习的神经网络，具体来说就是动作专家（Action Expert）——Transformer 中专门处理动作令牌的那一组权重参数（记作 θ）

Transformer 的输出中，只取对应 $H$ 个动作令牌的部分，通过线性投影解码为向量场：

$$v_\theta(A_t^\tau, o_t) = \text{LinearProj}(\text{TransformerOutput}_{[\text{action tokens}]})$$

> We only use the transformer outputs corresponding to the $H$ noisy actions, which are decoded into $v_\theta(A_t^\tau, o_t)$ using a linear projection.

(Black et al., 2026)

#### 3.3 流匹配积分

推理时，从纯噪声 $A_t^0 \sim \mathcal{N}(0, I)$ 出发，通过 10 步欧拉积分得到干净动作块：

$$A_t^{\tau+\delta} = A_t^\tau + \delta \cdot v_\theta(A_t^\tau, o_t), \quad \delta = 0.1$$

每一步都需要一次 Transformer 前向传播（复用缓存的观测 KV）。

#### 3.4 时间步采样分布（训练时）

训练时，流匹配时间步 $\tau$ 从**偏移 Beta 分布**中采样，强调低时间步（高噪声水平）：

$$p(\tau) = \text{Beta}\left(\frac{s-\tau}{s}; 1.5, 1\right), \quad s = 0.999$$

> We designed a timestep sampling distribution that emphasizes low timesteps (high noise levels); additionally, timesteps above a given threshold $s$ are not sampled at all.

(Black et al., 2026)

这与图像生成中的做法不同——因为机器人观测 $o_t$ 对动作的约束远强于文本标签对图像的约束，所以模型需要更多地在高噪声区域学习。

### 四、与传统方法的对比

#### 4.1 动作分块 vs. 自回归动作预测

| 特性 | 动作分块（π₀） | 自回归动作预测（如 OpenVLA） |
|------|---------------|---------------------------|
| 生成方式 | 一次性并行生成 $H$ 个动作 | 逐个生成，每个动作依赖前一个 |
| 推理频率 | 每 $H$ 步推理一次 | 每步推理一次 |
| 控制频率 | 高（50 Hz） | 低（受限于推理速度） |
| 误差累积 | 无（并行生成） | 有（每一步误差传递到下一步） |
| 动作分布建模 | 连续分布（流匹配） | 离散分布（分类头） |

> This allows us to handle high-frequency action chunks (up to 50 Hz) and highly dexterous tasks, which we show pose a major challenge for prior autoregressive VLAs.

(Black et al., 2026)

#### 4.2 电子信息工程视角

**动作分块架构可以类比为数字通信中的帧传输（frame transmission）：**

1. **帧（Frame）** = 动作块 $A_t$，包含 $H$ 个"采样点"（动作）
2. **帧率（Frame Rate）** = 推理频率（50 Hz 机器人每 0.5 秒推理一次）
3. **采样率（Sampling Rate）** = 控制频率（50 Hz）
4. **缓冲（Buffering）** = 一次性生成一帧，然后逐个输出采样点

这种"批处理"方式类似于音频处理中的**分帧加窗**——将连续的信号流分割为固定长度的帧，每帧独立处理，然后拼接输出。这样做的好处是：
- 充分利用并行计算（GPU 一次性处理 $H$ 个令牌）
- 降低推理延迟的约束（不需要实时推理）
- 帧内动作可以互相"协商"，保证时序平滑性

**与自回归方法的对比**：自回归动作预测类似于**串行通信**（如 UART）——每个比特（动作）依次传输，前一个比特的延迟会阻塞后续所有比特。而动作分块类似于**并行总线**（如 PCIe）——一帧数据同时传输，延迟只取决于帧长度而非比特数。

### 五、总结：动作分块的工作流程

```
时间步 t:
  1. 采集观测 o_t = [图像 I_t1...I_tn, 语言指令 l_t, 机器人状态 q_t]
  2. 编码图像 → 图像令牌
  3. 编码语言 + 状态 → 语言/状态令牌
  4. 缓存观测令牌的 KV（1 次前向传播，32 ms）
  5. 初始化噪声动作块 A_t^0 ~ N(0, I)
  6. 对 τ = 0, 0.1, 0.2, ..., 0.9（10 步流匹配）:
     a. 将 A_t^τ 编码为动作令牌嵌入（含 τ 编码）
     b. Transformer 前向传播（复用观测 KV，27 ms/10步）
     c. 解码输出 → 向量场 v_θ
     d. 欧拉积分: A_t^{τ+0.1} = A_t^τ + 0.1 · v_θ
  7. 得到干净动作块 A_t = [a_t, a_{t+1}, ..., a_{t+49}]
  8. 以 50 Hz 逐个执行动作 a_t, a_{t+1}, ...
  9. 执行 25 个动作后（0.5 秒），回到步骤 1
```

这种架构使得 π₀ 能够同时满足**高频控制**（50 Hz）和**高精度灵巧操作**（折叠衣物、组装盒子等）的需求，这是传统自回归 VLA 模型无法实现的。
