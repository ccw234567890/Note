# $π_0$: A Vision-Language-Action Flow Model for General Robot Control

## 令牌分配规则（Token Routing Mechanism）

π₀ 模型的核心架构创新在于：**所有令牌拼接成一个序列，但每个令牌根据其类型被路由到不同的专家权重**。以下是对这一机制的完整总结。

### 一、输入序列结构

输入序列是一个完整的拼接序列：

```
[It1, ..., Itn, lt, qt, a_t^τ, ..., a_{t+H-1}^τ]
```

| 块 | 内容 | 令牌数 |
|----|------|--------|
| 块 1 | 图像令牌 $I_{t1}...I_{tn}$ + 语言令牌 $l_t$ | 约数百个 |
| 块 2 | 本体状态 $q_t$ | 1 个 |
| 块 3 | 噪声动作令牌 $a_t^τ...a_{t+H-1}^τ$（H=50） | 50 个 |

所有令牌共享同一个位置编码和分块因果注意力掩码。

### 二、路由规则

每个令牌根据其类型被路由到不同的专家权重：

| 令牌类型 | 路由到 | 参数量 |
|---------|--------|--------|
| 图像令牌 $I_{t1}...I_{tn}$ | **VLM 骨干**（PaliGemma 初始化） | ~30 亿 |
| 语言令牌 $l_t$ | **VLM 骨干** | ~30 亿 |
| 状态令牌 $q_t$ | **动作专家（Action Expert）** | ~3 亿 |
| 动作令牌 $a_t^τ...a_{t+H-1}^τ$ | **动作专家** | ~3 亿 |

> The images and language prompt, [It1, ..., Itn, lt], are routed to the larger VLM backbone... The inputs not seen during VLM pretraining, [qt, Atτ], are routed to the action expert.
>
> (Black 等, 2026, 附录 B)

### 三、两个权重集的交互方式

> π0 is implemented as a single transformer with two sets of weights (also known as experts), where each token is routed to one of the experts; the weights interact only through the transformer's self-attention layers.
>
> (Black 等, 2026, 附录 B)

关键设计要点：

1. **FFN（前馈网络）是独立的**：VLM 令牌经过 VLM 骨干的 FFN，动作/状态令牌经过动作专家的 FFN
2. **自注意力是共享的**：所有令牌在同一个注意力层中互相计算注意力分数
3. **分块因果掩码**：块内双向注意力，块间因果注意力（未来块不能看到过去块）

### 四、计算流程可视化

```
输入序列: [I1...In, lt, qt, a1...a50]
              │      │    │    │
              │      │    │    │
           ┌──┴──────┴──┐ │    │
           │  路由决定   │ │    │
           └──┬──────┬──┘ │    │
              │      │    │    │
              ▼      ▼    ▼    ▼
         ┌─────────────┐ ┌──────────┐
         │ VLM 骨干    │ │ 动作专家  │
         │ (30亿参数)  │ │ (3亿参数) │
         │ FFN 层      │ │ FFN 层    │
         └──────┬──────┘ └─────┬────┘
                │              │
                └──────┬───────┘
                       │
                       ▼
              ┌─────────────────┐
              │ 共享自注意力层   │
              │ (所有令牌互相    │
              │  计算注意力)     │
              └─────────────────┘
                       │
                       ▼
              ┌─────────────────┐
              │ 输出: 只取动作   │
              │ 令牌位置的输出   │
              │ → 解码为 v_θ    │
              └─────────────────┘
```

### 五、为什么这样设计？

**1. 保护 VLM 预训练表示**

> The first block includes the input modalities from PaliGemma's VLM pre-training, which are prevented from attending to future blocks (which include new inputs) to minimize distribution shift from said pre-training.
>
> (Black 等, 2026, 附录 B)

图像和语言令牌在预训练时从未见过 $q_t$ 和动作令牌，所以让它们走独立的 FFN 权重，避免破坏预训练学到的表示。

**2. 加速推理**

> To speed up inference (which requires multiple forward passes of the action expert), we downsize the action expert to {width=1024, mlp dim=4096}, resulting in a parameter count of ∼300M.
>
> (Black 等, 2026, 附录 B)

Flow Matching 需要 10 步积分，每一步都要跑动作专家。如果把 30 亿参数的 VLM 骨干也一起跑 10 次，速度会非常慢。所以动作专家只有 3 亿参数，10 步积分只跑动作专家，而 VLM 骨干的 KV 缓存可以复用。

### 六、关键澄清

| 常见困惑 | 答案 |
|---------|------|
| "三个令牌都接在一起" | ✅ 是的，它们确实在同一个序列中，共享位置编码和注意力掩码 |
| "VLM 骨干只处理图像和语言" | ✅ 通过路由机制，图像/语言令牌的 FFN 走 VLM 骨干权重 |
| "动作专家处理状态和动作" | ✅ 状态/动作令牌的 FFN 走动作专家权重 |
| "序列被拆成两部分了吗？" | ❌ 不是物理拆分，而是每个令牌被路由到不同的专家权重，但自注意力层是共享的 |

这本质上是一个 **Mixture of Experts (MoE)** 架构，有两个专家：一个大的 VLM 专家（30 亿参数）和一个小的动作专家（3 亿参数），通过共享的自注意力层交互。
