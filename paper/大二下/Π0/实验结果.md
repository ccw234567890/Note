
# $\\pi_0$: A Vision-Language-Action Flow Model for General Robot Control

本文的实验部分围绕四个核心研究问题展开，分别对应 Fig.7、Fig.9、Fig.11、Fig.13 四张主图。以下逐一讲解每个图做了什么实验、结果是什么、结论是什么。

## Fig.7 — 开箱即用（Out-of-Box）零样本评估

### 实验内容

评估 π₀ **在预训练后、不经过任何微调**，直接通过语言指令执行任务的能力。对比基线包括：

| 方法 | 说明 |
|------|------|
| **π₀（full）** | 完整训练 700k 步 |
| **π₀（parity）** | 仅训练 160k 步（与基线计算量持平） |
| **π₀-small** | 无 VLM 初始化的小模型 |
| **OpenVLA** | 7B 参数 VLA 模型，在 π₀ 的混合数据上训练 |
| **OpenVLA（UR5e only）** | 仅在 UR5e 数据上微调的 OpenVLA |
| **Octo** | 93M 参数扩散模型 |

评估任务共 5 个：**Shirt Folding**（叠衬衫）、**Bussing Easy**（简单收拾餐桌）、**Bussing Hard**（困难收拾餐桌）、**Grocery Bagging**（装袋杂货）、**Toast out of Toaster**（从烤面包机取出吐司）。

### 结果

> Across all tasks and all comparisons, even the "parity" version of our model outperforms all baselines, and the full version of our model achieves the best results by a large margin.

(Black 等, 2026)

- **π₀（full）** 在所有任务上取得最佳成绩，叠衬衫和简单收拾餐桌接近完美
- **π₀（parity）**（仅 160k 步）仍然**超过所有基线**
- **π₀-small**（无 VLM）也超过了 OpenVLA 和 Octo
- OpenVLA 表现最差，因其自回归离散化架构不支持动作块（action chunks）
- Octo 支持动作块但表征能力有限

### 结论

1. **Flow Matching + VLM 骨干的组合至关重要**：大容量架构与连续动作分布建模能力缺一不可
2. **VLM 预训练带来显著提升**：π₀-small 虽小但优于 OpenVLA，说明架构设计本身就有优势，而完整 π₀ 更优则归功于 VLM 初始化
3. **π₀ 是目前最强的机器人基础模型**，开箱即用能力远超现有方法

## Fig.9 — 语言指令跟随能力评估

### 实验内容

评估 π₀ **微调后**的语言指令跟随能力，对比 π₀ 与 π₀-small（无 VLM 初始化）。评估条件分为 5 种：

| 条件 | 说明 |
|------|------|
| **π₀-flat / π₀-small-flat** | 仅接收整体任务指令（如"把杂货装袋"） |
| **π₀-human / π₀-small-human** | 接收人类专家提供的中间步骤指令 |
| **π₀-HL** | 接收高层 VLM 策略自动生成的中间指令 |

评估任务共 3 个：**Bussing**（收拾餐桌）、**Table Setting**（摆桌子）、**Grocery Bagging**（装袋杂货）。

### 结果

> The results show a significant improvement with π0 from intermediate language commands provided by a human expert and to a lesser degree by an autonomous high-level policy. Notably, due to π0-small's limited language following ability, overall it does not gain with the addition of a high-level expert.

(Black 等, 2026)

- **π₀ 的语言跟随准确率显著高于 π₀-small**
- π₀ 从人类专家中间指令中获益最大（π₀-human > π₀-HL > π₀-flat）
- π₀-HL（高层 VLM 自动指令）也有提升，但幅度小于人类专家
- **π₀-small 几乎无法从高层指令中获益**，因其语言理解能力有限

### 结论

1. **VLM 预训练大幅提升了语言指令跟随能力**，这是 π₀-small 无法比拟的
2. π₀ 的语言能力可以直接转化为**更好的自主任务执行表现**（配合高层 VLM 策略）
3. 人类专家提供的中间指令效果最好，但高层 VLM 策略也能实现**完全自主的**多步骤任务执行

## Fig.11 — 新灵巧任务的微调学习

### 实验内容

评估 π₀ 在**全新任务**上的微调学习能力，这些任务与预训练数据有不同程度的差异。任务分为"容易"和"困难"两个层级：

| 任务 | 层级 | 说明 |
|------|------|------|
| **Stack Bowls**（叠碗） | 容易 | 类似预训练中的收拾餐具 |
| **Towel Folding**（叠毛巾） | 容易 | 类似预训练中的叠衬衫 |
| **Tupperware in Microwave**（放微波炉） | 中等 | 容器操作类似预训练，但微波炉是新的 |
| **Paper Towel Replacement**（换纸巾） | 困难 | 全新物体和动作 |
| **Franka Items in Drawer**（放抽屉） | 困难 | 全新机器人平台和任务 |

对比基线包括：
- **OpenVLA** 和 **Octo**（使用公开预训练检查点 + 微调）
- **ACT** 和 **Diffusion Policy**（专门为灵巧操作设计的专用方法，从零训练）
- **π₀（scratch）**：从随机初始化在目标任务上训练（无预训练）

每种方法使用 **1h / 5h / 10h** 三种不同量的微调数据。

### 结果

> The results show that π0 generally outperforms other methods. Interestingly, the strongest prior models are the ones that are trained entirely from scratch on the target tasks, suggesting that leveraging pre-training in these domains presents a major challenge for prior approaches.

(Black 等, 2026)

- **π₀（fine-tune）** 在几乎所有任务和数据量下表现最佳
- **π₀（scratch）** 也优于 OpenVLA 和 Octo，说明 π₀ 架构本身就很强
- 最强的基线方法是 **ACT** 和 **Diffusion Policy**（从零训练），而非 OpenVLA 等预训练方法
- 在 Tupperware 任务上，π₀ 用 **1h 数据**就超过了基线用 5h 数据的效果
- 预训练带来的提升在**与预训练数据相似的任务**上最为显著（叠碗、叠毛巾），有时提升高达 **2 倍**

### 结论

1. **π₀ 的架构设计（Flow Matching + VLM）本身就优于现有方法**，即使从零训练也表现良好
2. **预训练带来额外增益**，尤其在数据量小、任务与预训练相似时效果最明显
3. 现有预训练方法（OpenVLA、Octo）在灵巧操作领域**未能有效利用预训练**，反而不如从零训练的专用方法
4. π₀ 在**数据效率**上具有显著优势（1h 数据达到基线 5h 的效果）

## Fig.13 — 复杂多阶段任务的掌握

### 实验内容

评估 π₀ 在**最具挑战性的多阶段任务**上的表现。这些任务需要 5~20 分钟完成，涉及数十个子行为。任务包括：

| 任务 | 是否在预训练中 | 难度说明 |
|------|---------------|----------|
| **Laundry Folding**（叠衣物） | ✅ 是 | 从随机揉成一团的篮子中取出衣物并折叠 |
| **Mobile Laundry**（移动叠衣） | ✅ 是 | 移动机器人在不同位置叠衣物 |
| **Dryer Unloading**（取干衣） | ✅ 是 | 从烘干机取出衣物放入篮子 |
| **Table Bussing**（收拾餐桌） | ❌ 否 | 处理全新物体、密集杂乱场景 |
| **Box Building**（组装纸箱） | ❌ 否 | 将扁平纸箱折叠成型，需双臂协作 |
| **To-go Box**（打包餐盒） | ❌ 否 | 将食物装入餐盒并合盖 |
| **Packing Eggs**（装鸡蛋） | ❌ 否 | 将 6 个鸡蛋放入蛋盒并合盖 |

由于这些任务极难，其他方法无法解决，因此仅对比 π₀ 的三种消融变体：
- **π₀（fine-tune）**：预训练 + 微调（完整方案）
- **π₀（out-of-box）**：仅预训练，零样本
- **π₀（scratch）**：无预训练，从零训练

评分标准：1.0 = 完美执行，部分完成得部分分数。

### 结果

> The results show that π0 can solve many of these tasks, with our full pre-training and fine-tuning recipe performing best across the board. Note that many of these more difficult tasks show a very large improvement from using the pre-trained model, indicating that pre-training is especially useful with harder tasks.

(Black 等, 2026)

- **π₀（fine-tune）** 在所有任务上取得最佳成绩，平均超过最大分数的 **50%**
- **π₀（out-of-box）** 在预训练中出现过的任务上表现良好，但在新任务上受限
- **π₀（scratch）** 在困难任务上表现最差
- 预训练带来的提升在**最困难的任务**上最为显著（收拾餐桌、组装纸箱等）
- 不同任务间的绝对性能差异较大，取决于任务难度和在预训练中的覆盖程度

### 结论

1. **π₀ 在极具挑战性的灵巧操作任务上达到了新的 SOTA**，这是此前任何方法都未能实现的
2. **完整方案（预训练 + 微调）效果最佳**，两者缺一不可
3. **预训练对困难任务的帮助最大**——任务越难，预训练的增益越明显
4. 这些结果证明了**通用机器人基础模型范式的可行性**：预训练通用技能 → 微调适配特定任务
