# DeepSeekMath: Pushing the Limits of Mathematical Reasoning in Open Language Models

## 策略网络（Policy Network / Actor）与价值网络（Value Network / Critic）详解

## 一、基本概念

### 策略网络（Policy Network）$\pi_\theta$

**策略网络就是语言模型本身**。在 RLHF 的语境中，策略网络就是那个正在被训练的 LLM。

它的作用是：给定输入 $q$（问题/提示），输出一个 token 序列 $o$（回答）的概率分布：

$$\pi_\theta(o_t | q, o_{<t})$$

即：在给定问题 $q$ 和已生成的前 $t-1$ 个 token 的条件下，第 $t$ 个 token 是 $o_t$ 的概率。

**训练目标**：最大化奖励模型给出的评分，同时不偏离参考策略太远。

### 价值网络（Value Network / Critic）$V_\psi$

价值网络是一个**辅助模型**，它估计"当前状态有多好"——即从当前状态开始，未来能获得的**期望累积奖励**：

$$V_\psi(s_t) = \mathbb{E}\left[ \sum_{l=0}^{\infty} \gamma^l r_{t+l} \;\big|\; s_t \right]$$

在 LLM 语境中，"状态" $s_t$ 就是已生成的上下文 $(q, o_{<t})$，"奖励" $r_t$ 通常只在最后一个 token 给出（来自奖励模型）。

**价值网络的作用**：帮助计算 **Advantage（优势函数）**，衡量"当前动作相对于平均水平有多好"。

## 二、两者的关系：Actor-Critic 架构

PPO 使用的 Actor-Critic 架构中，两个网络协同工作：

```
                    ┌─────────────────────┐
                    │   输入：问题 q       │
                    └──────────┬──────────┘
                               │
                    ┌──────────▼──────────┐
                    │   策略网络 π_θ       │ ← 生成 token（Actor）
                    │   (语言模型本身)      │
                    └──────────┬──────────┘
                               │
                    ┌──────────▼──────────┐
                    │   输出 o = {o₁...oₙ} │
                    └──────────┬──────────┘
                               │
              ┌────────────────┼────────────────┐
              │                │                │
    ┌─────────▼─────────┐    ...    ┌──────────▼──────────┐
    │ 奖励模型 r_φ       │          │ 价值网络 V_ψ        │
    │ 给完整输出打分     │          │ 估计每个状态的价值   │
    └─────────┬─────────┘          └──────────┬──────────┘
              │                               │
              └──────────┬────────────────────┘
                         │
              ┌──────────▼──────────┐
              │  Advantage 计算     │
              │  A_t = r_t + γV(s_{t+1}) - V(s_t) │
              └──────────┬──────────┘
                         │
              ┌──────────▼──────────┐
              │  更新策略网络 π_θ    │
              │  (PPO 裁剪目标)      │
              └─────────────────────┘
```

### 具体流程

1. **策略网络** $\pi_\theta$ 生成回答 $o$
2. **奖励模型** $r_\phi$ 给回答打分
3. **价值网络** $V_\psi$ 估计每个 token 位置的状态价值
4. 用 GAE 计算 Advantage $A_t$：
   $$\hat{A}_t = \sum_{l=0}^{\infty} (\gamma\lambda)^l \delta_{t+l}, \quad \delta_t = r_t + \gamma V(s_{t+1}) - V(s_t)$$
5. 用 PPO 裁剪目标更新策略网络 $\pi_\theta$
6. 同时更新价值网络 $V_\psi$（最小化 TD 误差）

## 三、一个直观的例子

假设模型在解数学题，生成过程是逐步的：

| 步骤 | 已生成的 token | 策略网络做的事 | 价值网络做的事 |
|:----:|:--------------|:--------------|:--------------|
| t=1 | "Let x =" | 预测下一个 token 的概率分布 | 估计"当前状态"的价值 |
| t=2 | "Let x = 5" | 继续预测 | 估计新状态的价值 |
| ... | ... | ... | ... |
| t=n | 完整解答 | 生成完毕 | 给出最终状态价值 |

**价值网络在每个步骤都输出一个数值**，比如：
- 在 t=1 时，$V(s_1) = 0.3$（"刚开始，还看不出好坏"）
- 在 t=5 时，$V(s_5) = 0.7$（"推理方向似乎正确"）
- 在 t=10 时，$V(s_{10}) = 0.2$（"好像走偏了"）

然后 Advantage $A_t = r_t + \gamma V(s_{t+1}) - V(s_t)$ 就衡量了"这一步比预期好还是差"。

## 四、PPO 中价值网络的问题

### 问题 1：显存开销巨大

价值网络通常和策略网络规模相当。在 PPO 中，你需要同时加载：

| 模型 | 参数量（以 7B 模型为例） |
|:----|:---------------------:|
| 策略网络 $\pi_\theta$ | 7B |
| 价值网络 $V_\psi$ | 7B |
| 奖励模型 $r_\phi$ | 7B |
| 参考模型 $\pi_{\text{ref}}$ | 7B |
| **总计** | **28B 参数** |

> 论文原话："As the value function employed in PPO is typically another model of comparable size as the policy model, it brings a substantial memory and computational burden."

### 问题 2：训练困难

在 LLM 场景中，奖励模型通常只给**最后一个 token** 打分（即完整回答的好坏），但价值网络需要在**每个 token 位置**都输出一个价值估计。这意味着价值网络必须从稀疏的奖励信号中学习推断中间状态的价值——这是一个非常困难的任务。

### 问题 3：额外的误差源

价值网络的估计本身就有误差，这个误差会传播到 Advantage 的计算中，进而影响策略网络的更新质量。

---

## 五、GRPO 的解决方案：彻底去掉价值网络

GRPO 的核心创新就是**完全抛弃价值网络**，改用**组内归一化**来计算 Advantage：

```
PPO:   策略网络 → 采样1个输出 → 奖励模型 → 价值网络 → GAE → Advantage
GRPO:  策略网络 → 采样G个输出 → 奖励模型 → 组内归一化 → Advantage
```

### 组内归一化的 Advantage

对于每个问题 $q$，采样 $G$ 个输出 $\{o_1, o_2, \cdots, o_G\}$，奖励模型打分得到 $\{r_1, r_2, \cdots, r_G\}$：

$$\hat{A}_{i,t} = \frac{r_i - \text{mean}(\mathbf{r})}{\text{std}(\mathbf{r})}, \quad \forall t$$

**不需要价值网络**，只需要同一问题下多个输出的相对比较。

### 为什么这有效？

奖励模型本身就是通过**比较同一问题的不同输出**来训练的（pairwise preference）。GRPO 的组内 Advantage 与奖励模型的训练范式**天然一致**——它问的是"这个输出在组内排第几？"，而不是"这个输出的绝对分数是多少？"。

## 六、总结对比

| 方面 | 策略网络（Actor） | 价值网络（Critic） |
|:----|:----------------|:-----------------|
| **是什么** | 语言模型本身 $\pi_\theta$ | 辅助模型 $V_\psi$ |
| **作用** | 生成 token，决定"做什么" | 估计状态价值，决定"做得多好" |
| **PPO 中** | ✅ 需要，被优化 | ✅ 需要，辅助计算 Advantage |
| **GRPO 中** | ✅ 需要，被优化 | ❌ **不需要**，用组内归一化替代 |
| **参数量** | 7B（以 DeepSeekMath 为例） | 7B（与策略网络等大） |
| **训练难度** | 正常 | 困难（从稀疏奖励中学习密集价值） |

**一句话总结**：策略网络是**做决策的**（生成文本），价值网络是**做评估的**（估计状态好坏）——PPO 两个都需要，GRPO 通过组内比较巧妙地甩掉了价值网络这个"包袱"。
