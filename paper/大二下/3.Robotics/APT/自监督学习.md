## APT 中的自监督训练（Self-Supervised Pretraining）是什么？

### 一、先理解：什么是"自监督学习"？

**自监督学习（Self-Supervised Learning）** 是一种不需要人工标注标签的训练方式。模型从数据本身构造"伪标签"来学习有用的表示。

#### 生活类比：自学画画

|学习方式|做法|例子|
|---|---|---|
|**监督学习**|老师给你标注好的样本|"这是猫，这是狗" → 你学习区分猫和狗|
|**自监督学习**|你自己从数据中找规律|你拿1000张画，自己发现"圆形的东西通常是太阳，蓝色区域通常是天空"|

自监督学习的核心是：**不需要人工标注，数据本身就是自己的老师。**

### 二、APT 为什么需要自监督训练？

论文原文：

> _"APT learns timestamp-aware affine parameters via self-supervised pretraining, while freezing backbone and normalization."_

**冻结 backbone 和归一化模块**，只训练 APT 的参数。

#### 为什么不能直接一起训练？

如果 APT 和 backbone 一起从头训练，会出现两个问题：

|问题|表现|后果|
|---|---|---|
|**梯度冲突**|backbone 想学时序模式，APT 想学分布信息|两者互相干扰|
|**冷启动**|刚开始时，原型向量是随机初始化的|给 backbone 提供错误的分布信息，导致训练不稳定|

**解决方案**：先冻结 backbone，让 APT 单独通过自监督方式"预热"（warm-up），学到合理的时间戳表示和原型向量，再联合优化。

### 三、APT 自监督训练的三个损失函数

论文公式(10)-(12)定义了三个自监督损失：

#### 损失1：正交损失（公式10）

$$\mathcal{L}_{orth} = ||E^\top E \odot (1 - I)||^2_2 + ||I - E^\top E \odot I||^2_2$$

其中 $E = \{T^{TiD}, T^{DiW}, P\}$，即所有时间戳嵌入和原型向量的集合。

**目的**：让嵌入向量之间彼此正交（不相关）

| 项                          | 含义  | 作用  |                     |              |
| -------------------------- | --- | --- | ------------------- | ------------ |
| $E^top E odot (1 - I)^2_2$ |     |     | 惩罚非对角元素（不同向量之间的相似度） | 让不同嵌入互不相关    |
| $I - E^top E odot I^2_2$   |     |     | 惩罚对角元素偏离1           | 让每个嵌入向量的范数为1 |


**直观理解**：就像要求调色板上的每种基础色都是纯色，不能混合——红色就是红色，蓝色就是蓝色，不能红里透蓝。

论文引用 Barlow Twins 方法，这是一种经典的自监督学习框架。

#### 损失2：负载均衡损失（公式11）

$$\mathcal{L}_{balance} = \sum_{j=1}^N \left( \frac{\sum_{i=1}^B W_{i,j}}{\sum_{i=1}^B \sum_{j=1}^N W_{i,j}} - \frac{1}{N} \right)^2$$

其中 $W \in \mathbb{R}^{B \times N}$ 是批次中所有样本的原型权重矩阵。

**目的**：防止某些原型被过度使用，而其他原型"饿死"

**直观理解**：一个班级里有40个原型（学生），如果每次回答问题都是同3个人举手，其他37个人从不举手——这不健康。负载均衡损失强制所有原型被**均匀使用**。

**例子**：假设 N=5 个原型，一个批次 B=32 个样本：

|原型|被使用的次数|理想比例|实际比例|惩罚|
|---|---|---|---|---|
|原型1|28次|20%|**35%**|❌ 过高|
|原型2|25次|20%|**31%**|❌ 过高|
|原型3|8次|20%|10%|✅ 接近|
|原型4|5次|20%|6%|✅ 接近|
|原型5|14次|20%|18%|✅ 接近|

原型1和2被过度使用，损失函数会惩罚这种不平衡。

#### 损失3：仿射正则化损失（公式12）

以 $\gamma$ 为例：

$$\mathcal{L}_R = \frac{1}{B}\left(1 - \sqrt{\sum_{i=1}^B \gamma_i^2}\right)^2 + \frac{1}{B}\left(\sum_{i=1}^B \gamma_i\right)^2$$

**目的**：防止仿射参数 $\gamma, \beta$ 变得过大或过小，保持稳定

|项|含义|作用|
|---|---|---|
|$frac{1}{B}(1 - sqrt{sum gamma_i^2})^2$|惩罚 $gamma$ 的 L2 范数偏离1|让 $gamma$ 的均方根接近1|
|$frac{1}{B}(sum gamma_i)^2$|惩罚 $gamma$ 的均值偏离0|让 $gamma$ 的均值接近0|

**直观理解**：就像给仿射参数加了一个"软约束"——你可以调整，但不能太离谱。

#### 总损失（公式13）

$$\mathcal{L}_{APT} = \mathcal{L}_{orth} + \mathcal{L}_{balance} + \mathcal{L}_R$$

这三个损失**都不需要任何标签**，完全从数据自身的结构出发，所以叫"自监督"。

### 四、两阶段训练流程

#### 阶段一：自监督预训练

```
冻结 backbone ❌  ← backbone 不更新
冻结归一化 ❌    ← 归一化不更新
训练 APT ✅      ← 只更新 APT 的参数（时间戳嵌入 + 原型向量 + MLP）
损失: L_APT = L_orth + L_balance + L_R
学习率: 5e-5（固定，比 backbone 的学习率小很多）
训练轮数: 1~5 个额外 epoch（直到收敛）
```

#### 阶段二：联合优化

```
训练 backbone ✅  ← backbone 正常更新
训练归一化 ✅    ← 归一化正常更新
训练 APT ✅      ← APT 继续更新
损失: L_main = L_normal + L_MSE
```

### 五、为什么自监督预训练有效？

论文通过消融实验验证了自监督预训练的必要性：

|变体|性能变化|原因|
|---|---|---|
|**W/O Loss_APT**（跳过预训练）|**大幅下降** ❌|APT 冷启动，与 backbone 互相干扰|
|**W/O λ**（无损失权重）|**下降** ❌|APT 共享 backbone 的大学习率，导致过拟合|
|**W/O Loss_orth**（无正交损失）|**下降** ❌|嵌入向量之间高度相关，信息冗余|
|**W/O Loss_balance**（无负载均衡）|**下降** ❌|少数原型被过度使用，其他原型浪费|
|**W/O Loss_R**（无仿射正则化）|**下降** ❌|仿射参数不稳定，训练震荡|
|**完整 APT**|**最佳** ✅|三个损失协同工作|

论文原文：

> _"Without λ, APT shares the backbone's rate, causing overfitting and unstable timestamp semantics."_

> _"W/O Loss_APT: Skip pretraining entirely, optimizing APT only with the main loss."_ → 性能大幅下降

### 六、自监督 vs 监督 vs 无监督：一张表看懂

|维度|**监督学习**|**自监督学习**|**无监督学习**|
|---|---|---|---|
|**是否需要标签**|✅ 需要人工标注|❌ 不需要|❌ 不需要|
|**学习目标**|拟合标签|从数据自身构造"伪任务"|发现数据内在结构|
|**APT 中的角色**|阶段二的 MSE 损失|阶段一的三个损失|—|
|**例子**|告诉模型"这个序列的下一段是 X"|让原型向量彼此正交、均匀使用|聚类（但 APT 不是纯无监督）|

### 一句话总结

> **APT 的自监督训练就是在冻结 backbone 的情况下，用三个不需要标签的损失（正交损失让嵌入多样化、负载均衡损失让原型被均匀使用、仿射正则化损失让参数稳定）先让 APT 自己"热身"学到合理的时间戳表示，然后再与 backbone 联合优化。这就像让一个新员工先自学公司规章制度（自监督预训练），再开始实际干活（联合优化）——比直接上岗效率高得多。**
