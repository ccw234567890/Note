
# $π_{0.7}$: a Steerable Generalist Robotic Foundation Model with Emergent Capabilities

## 评估数据（Evaluation Data）≠ 片段元数据（Episode Metadata）

### 评估数据是什么？

评估数据是 **π₀*.6 RL 专家在训练和评估过程中收集的完整轨迹数据**，包括：

> "data collected by the π0∗.6 model during RL training as additional examples"
> (Intelligence 等, 2026)

具体来说，评估数据包含：
- **RL 训练过程中的轨迹**：π₀*.6 在强化学习训练时反复试错产生的所有数据
- **成功和失败的轨迹**：既有成功完成任务的，也有中途失败的
- **次优轨迹**：虽然完成了任务但执行质量不高的

**这些数据是"原始数据"**——包含完整的观测序列、动作序列、状态信息等。

### 片段元数据是什么？

片段元数据是**附加在每条评估数据上的标签/标注**，用来描述这条轨迹的质量和特性：

| 元数据字段             | 含义                  | 示例           |
| ----------------- | ------------------- | ------------ |
| **Speed（速度）**     | 片段的总时间步数，离散化为500步区间 | "8000 steps" |
| **Quality（质量）**   | 1-5分的任务执行质量评分       | "5/5"        |
| **Mistake（错误标记）** | 是否在某个动作段中犯错         | "false"      |

### 两者的关系

```
评估数据（Evaluation Data）= 完整的轨迹数据（观测+动作+状态）
                                      ↓ 加上标注
片段元数据（Episode Metadata）= 贴在每条轨迹上的标签（速度、质量、错误）
                                      ↓ 一起输入模型
π₀.₇ 的完整训练样本 = 评估数据 + 片段元数据
```

**打个比方**：
- **评估数据** = 学生的考试答卷（完整的答题过程）
- **片段元数据** = 老师在答卷上批的分数和评语（质量评分、用时、错误标记）
- π₀.₇ 不仅看答卷，还看分数评语，从而学会区分"好答卷"和"差答卷"

## 什么是知识蒸馏（Knowledge Distillation）？

### 核心定义

知识蒸馏是一种**将"教师模型"的知识迁移到"学生模型"**的技术。在 π₀.₇ 中：

> "This corresponds to a kind of 'distillation' process, where the generalist π0.7 model can inherit the capabilities of RL-trained specialists."
> (Intelligence 等, 2026)

### π₀.₇ 中的蒸馏过程

```
┌─────────────────────────────────────────────────────────┐
│                   知识蒸馏流程                            │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  Step 1: RL专家（教师）在真实环境中反复执行任务             │
│          ↓ 产生大量评估数据                               │
│  Step 2: 收集这些数据（成功+失败+次优）                    │
│          ↓ 加上元数据标注                                 │
│  Step 3: π₀.₇（学生）用这些数据训练                       │
│          ↓ 通过元数据区分"好行为"和"坏行为"                │
│  Step 4: π₀.₇ 继承了RL专家的能力                          │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

### 传统蒸馏 vs π₀.₇ 的蒸馏

| 对比维度 | 传统知识蒸馏 | π₀.₇ 的蒸馏 |
|---------|------------|------------|
| **教师模型** | 大模型（如 GPT-4） | **π₀*.6 RL 专家**（每个任务一个专用模型） |
| **学生模型** | 小模型（如 DistilBERT） | **π₀.₇ 通用模型**（一个模型处理所有任务） |
| **迁移方式** | 用教师的**输出概率**训练学生 | 用教师的**评估数据轨迹**训练学生 |
| **关键机制** | 模仿教师的软标签 | **元数据标注**让模型自己区分好坏 |
| **优势** | 学生更小更快 | 学生**一个模型匹配所有专家** |

### 为什么这很巧妙？

传统方法中，如果你想用 RL 专家的数据训练通用模型，会遇到一个大问题：

> **RL 专家的数据包含大量失败和次优轨迹**——如果直接全部喂给模型，模型反而会学到错误的行为。

π₀.₇ 的解决方案是：**用元数据标注告诉模型每条数据的质量**。

> "Incorporating the episode metadata into the context allows our model to effectively use all of this evaluation data and... enables it to attain similar performance as models that are specialized for high performance on individual tasks with RL."
> (Intelligence 等, 2026)

### 直观类比

| 角色 | 类比 | 说明 |
|------|------|------|
| **RL 专家** | 🏆 **奥运冠军** | 专门练一个项目，水平极高 |
| **评估数据** | 📹 **奥运冠军的训练录像** | 记录了冠军的训练过程（包括失误） |
| **片段元数据** | 🏷️ **录像上的标签** | "这一跳完美（质量5/5）"、"这一跳失误（错误标记=true）" |
| **知识蒸馏** | 📚 **看录像学习** | 普通运动员看冠军的训练录像，通过标签区分好动作和坏动作 |
| **π₀.₇** | 🏅 **十项全能运动员** | 看了所有项目的冠军录像，学会所有项目 |

### 总结

| 问题 | 答案 |
|------|------|
| **评估数据 = 片段元数据？** | **不是。** 评估数据是**完整的轨迹数据**，片段元数据是贴在轨迹上的**质量标签** |
| **两者的关系？** | 评估数据 + 片段元数据 = π₀.₇ 的完整训练样本 |
| **知识蒸馏是什么？** | 将 RL 专家的能力**通过其评估数据**迁移到 π₀.₇ 通用模型的过程 |
| **蒸馏的关键机制？** | **元数据标注**让模型能从混合质量数据中学习，区分好行为和坏行为 |
| **蒸馏的结果？** | π₀.₇ **一个模型匹配所有 RL 专家**，无需为每个任务单独微调 |
