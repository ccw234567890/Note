## 一、双 Q 网络 — 两个独立的 Critic

### 先看 SAC 论文怎么说

SAC 论文附录 C（算法细节）里明确写了：

> We use **two Q-functions** to mitigate positive bias in the policy improvement step that is known to degrade performance of value-based methods.

(Haarnoja 等, 2019, Appendix C)

### 为什么需要两个 Q 网络？

#### 问题：Q 学习的过估计（Overestimation Bias）

普通 Q 学习在算目标值时：

$$y = r + \gamma \max_{a'} Q(s', a')$$

这个 $\max$ 操作有一个问题：**如果 Q 网络对某个动作的估值偏高（噪声导致的），$\max$ 会选到这个被高估的动作**。

举个例子：

|动作|真实 Q 值|Q 网络估计值|
|---|---|---|
|向左走|5.0|5.1（偏高 0.1）|
|向右走|5.0|4.9（偏低 0.1）|
|跳起来|3.0|5.0（**偏高 2.0**）|

$\max$ 会选"跳起来"（5.0），但实际上它只有 3.0。**Q 值被系统性高估**，而且会随着训练不断累积放大。

#### SAC 的解决方案：取两个 Q 的最小值

SAC 训练**两个独立的 Q 网络**：$Q_{\theta_1}$ 和 $Q_{\theta_2}$，参数不同、初始化不同。

计算目标值时：

$$y = r + \gamma \left( \min_{i=1,2} Q_{\bar{\theta}_i}(s_{t+1}, \tilde{a}_{t+1}) - \alpha \log \pi(\tilde{a}_{t+1}|s_{t+1}) \right)$$

**取两个 Q 中较小的那个。**

#### 为什么取最小值能解决过估计？

|情况|$Q_{theta_1}$|$Q_{theta_2}$|$min(Q_1, Q_2)$|
|---|---|---|---|
|都准确|5.0|5.0|**5.0** ✅|
|$Q_1$ 偏高，$Q_2$ 准确|6.0|5.0|**5.0** ✅ 取到准确的|
|$Q_1$ 偏低，$Q_2$ 准确|4.0|5.0|**4.0** ❌ 偏低了一点|
|都偏高|6.0|5.5|**5.5** ✅ 取到较小的偏高|

**关键洞察：** 两个独立的 Q 网络用不同的随机初始化，它们的噪声是**不相关的**。同时被高估的概率远低于单个网络被高估的概率。取最小值虽然可能轻微低估，但**低估比高估安全**——低估不会让策略误以为某个坏动作很好。

#### 训练时的细节

```
每个训练步骤：
  1. 从经验池采样 (s, a, r, s')
  
  2. 用策略采样下一个动作：ã' ~ π_φ(s')
  
  3. 算两个目标 Q 值：
     y₁ = r + γ(min(Q_θ̄₁(s',ã'), Q_θ̄₂(s',ã')) - α log π(ã'|s'))
     y₂ = r + γ(min(Q_θ̄₁(s',ã'), Q_θ̄₂(s',ã')) - α log π(ã'|s'))
     ← 注意：y₁ = y₂，因为 min 是同一个值
  
  4. 分别更新两个 Q 网络：
     J_Q(θ₁) = ½(Q_θ₁(s,a) - y)²  → 梯度更新 θ₁
     J_Q(θ₂) = ½(Q_θ₂(s,a) - y)²  → 梯度更新 θ₂
  
  5. 两个目标 Q 网络各自 EMA 软更新：
     θ̄₁ ← τθ₁ + (1-τ)θ̄₁
     θ̄₂ ← τθ₂ + (1-τ)θ̄₂
  
  6. Actor 损失用较小的 Q：
     J_π(φ) = E[α log π_φ(a|s) - min(Q_θ₁(s,a), Q_θ₂(s,a))]
```

## 二、目标 Q 网络 — EMA 软更新

### 这两个 Q 到底是什么？

现在把所有 Q 网络理清楚：

|符号|名称|数量|用途|更新方式|
|---|---|---|---|---|
|$Q_{theta_1}$|**在线 Q 网络 1**|1个|算当前 Q 值，参与 Actor 损失|梯度下降|
|$Q_{theta_2}$|**在线 Q 网络 2**|1个|同上，独立参数|梯度下降|
|$Q_{bar{theta}_1}$|**目标 Q 网络 1**|1个|算目标值 $y$ 中的 $Q_{bar{theta}}$|EMA 软更新|
|$Q_{bar{theta}_2}$|**目标 Q 网络 2**|1个|同上|EMA 软更新|

**总共 4 个 Q 网络！**（2个在线 + 2个目标）

### EMA 软更新怎么工作？

论文公式(7)：

$$\bar{\theta}_i \leftarrow \tau \theta_i + (1 - \tau) \bar{\theta}_i, \quad i=1,2$$

其中 $\tau = 0.005$（默认值）。

#### 具体数值例子

假设 $\tau = 0.005$：

|步骤|在线 $theta$|目标 $bar{theta}$ 更新公式|目标 $bar{theta}$ 新值|
|---|---|---|---|
|初始|1.000|—|1.000（复制在线）|
|第1步|1.100|$0.005 times 1.100 + 0.995 times 1.000$|**1.0005**（只动了 0.05%）|
|第10步|1.500|$0.005 times 1.500 + 0.995 times 1.025$|**1.027**（缓慢跟进）|
|第100步|2.000|$0.005 times 2.000 + 0.995 times 1.250$|**1.254**（远远落后）|

**目标网络永远追不上在线网络，这正是设计目的！** 它提供一个**稳定的目标值**。

## 三、未来的回报是用熵来算的吗？

**是的！这正是 SAC 的核心创新。**

### 普通 Q 学习的目标值

$$y_{\text{标准}} = r + \gamma \max_{a'} Q(s', a')$$

未来回报 = **纯奖励的折现和**

### SAC 的目标值

$$y_{\text{SAC}} = r + \gamma \left( \min_{i=1,2} Q_{\bar{\theta}_i}(s', \tilde{a}') - \alpha \log \pi(\tilde{a}'|s') \right)$$

未来回报 = **奖励的折现和 + 熵的折现和**

#### 展开看看

SAC 的最大熵目标（论文公式1）：

$$J(\pi) = \sum_{t=0}^T \mathbb{E}_{(s_t,a_t)\sim\rho_\pi} \left[ r(s_t,a_t) + \alpha \mathcal{H}(\pi(\cdot|s_t)) \right]$$

展开：

$$J(\pi) = \mathbb{E}\left[ \underbrace{r(s_0,a_0) + \gamma r(s_1,a_1) + \gamma^2 r(s_2,a_2) + \cdots}_{\text{标准回报}} + \alpha \underbrace{(\mathcal{H}(\pi_0) + \gamma \mathcal{H}(\pi_1) + \gamma^2 \mathcal{H}(\pi_2) + \cdots)}_{\text{熵的折现和}} \right]$$

#### 所以 Q 函数也在算熵

论文公式(2)定义的 Soft Bellman 算子：

$$T^\pi Q(s_t, a_t) \triangleq r(s_t, a_t) + \gamma \mathbb{E}_{s_{t+1}\sim p} [V(s_{t+1})]$$

其中（公式3）：

$$V(s_t) = \mathbb{E}_{a_t\sim\pi} [Q(s_t,a_t) - \alpha \log \pi(a_t|s_t)]$$

**$V(s)$ 里包含了 $-\alpha\log\pi$（熵奖励），所以 Q 函数递归地包含了所有未来时间步的熵。**

#### 直观理解

> SAC 的 Q 函数回答的问题是： **"在当前状态做这个动作，未来能拿到多少奖励 + 多少熵？"**

|场景|标准 Q 值|SAC 的 Soft Q 值|
|---|---|---|
|确定性策略，高奖励|高|高（但熵低，可能扣分）|
|随机策略，中等奖励|中|可能更高（熵奖励加分）|
|确定性策略，低奖励|低|很低（奖励低 + 没探索）|

## 五、一句话总结

|问题|答案|
|---|---|
|**双 Q 网络是什么？**|两个独立初始化的 Critic，取 $min(Q_1, Q_2)$ 作为目标值，防止 Q 值被高估|
|**目标 Q 网络是什么？**|在线 Q 网络的"慢速影子"，通过 EMA（$bar{theta} leftarrow tautheta + (1-tau)bar{theta}$）缓慢更新，提供稳定的目标值|
|**未来回报用熵算吗？**|**是的！** SAC 的 Q 函数递归包含 $-alphalogpi$（熵奖励），所以 Soft Q 值 = 未来奖励的折现和 + 未来熵的折现和|
好问题！从 SAC 论文附录 C 出发，给你讲清楚两种 Q 网络的输入输出。

### 两种 Q 网络的结构

#### 在线 Q 网络 $Q_{\theta_1}, Q_{\theta_2}$

||输入|输出|
|---|---|---|
|**训练时**|$(s_t, a_t)$ — 状态和动作的**拼接向量**|一个标量 $Q_{theta_i}(s_t, a_t)$|
|**推理时**|$(s_t, a_t)$|一个标量|

#### 目标 Q 网络 $Q_{\bar{\theta}_1}, Q_{\bar{\theta}_2}$

||输入|输出|
|---|---|---|
|**训练时**|$(s_{t+1}, tilde{a}_{t+1})$ — 下一步状态 + 策略采样的动作|一个标量 $Q_{bar{theta}_i}(s_{t+1}, tilde{a}_{t+1})$|
|**推理时**|不用（只在训练中算目标值 $y$）|—|

### 具体网络结构（论文附录 C）

论文附录 C 明确写了 SAC 的网络架构：

> The function approximator for the Q-function is a **2-layer MLP with 256 units** and ReLU non-linearities.

(Haarnoja 等, 2019, Appendix C)

```
输入层: [s_dim + a_dim]  ← 状态向量和动作向量拼接
    │
    ▼
全连接层 1: 256 个神经元 + ReLU
    │
    ▼
全连接层 2: 256 个神经元 + ReLU
    │
    ▼
输出层: 1 个神经元（无激活函数）→ 标量 Q 值
```

### 数值例子

假设：

- 状态维度 $s_{\text{dim}} = 3$（位置、速度、角度）
- 动作维度 $a_{\text{dim}} = 1$（力矩）

#### 在线 Q 网络

```
输入: s = [0.5, 1.2, -0.3]    ← 3 维
      a = [0.8]               ← 1 维
      └────────┬────────┘
               ▼
      拼接: [0.5, 1.2, -0.3, 0.8]  ← 4 维
               │
               ▼
      MLP (256 → 256 → 1)
               │
               ▼
      输出: Q_θ(s, a) = 5.2  ← 1 个标量
```

#### 目标 Q 网络（训练时）

```
输入: s' = [0.6, 1.1, -0.2]   ← 下一步状态
      ã' = [0.5]              ← 策略采样的下一步动作
      └────────┬────────┘
               ▼
      拼接: [0.6, 1.1, -0.2, 0.5]  ← 4 维
               │
               ▼
      MLP (256 → 256 → 1)
               │
               ▼
      输出: Q_θ̄(s', ã') = 4.8  ← 1 个标量
```

### 关键区别总结

|对比维度|在线 Q 网络 $Q_theta$|目标 Q 网络 $Q_{bar{theta}}$|
|---|---|---|
|**输入**|$(s_t, a_t)$ — 当前状态和当前动作|$(s_{t+1}, tilde{a}_{t+1})$ — 下一步状态和策略采样的动作|
|**输出**|标量 Q 值|标量 Q 值|
|**网络结构**|完全一样（2层 MLP, 256 units）|完全一样（复制在线网络）|
|**参数更新**|梯度下降（每个 step 更新）|EMA 软更新（$tau=0.005$，缓慢跟进）|
|**用途**|① 算 Q 损失 ② 算 Actor 损失|只用来算目标值 $y$|
|**数量**|2 个（独立初始化）|2 个（各自对应一个在线网络）|

**一句话：** 四个 Q 网络结构完全一样（输入是状态+动作的拼接，输出是一个标量），只是参数不同、更新方式不同、输入的数据不同。在线 Q 网络用 $(s_t, a_t)$ 做预测，目标 Q 网络用 $(s_{t+1}, \tilde{a}_{t+1})$ 帮构造稳定的"正确答案" $y$。
