# PPGFlowECG: Latent Rectified Flow with Cross-Modal Encoding for PPG-Guided ECG Generation and Cardiovascular Disease Detection

## 速度场 $v_\theta$ 是怎么从 $z_{ppg}$、$z_{ecg}$ 和高斯噪声中学出来的？

这是一个**三步走**的过程，每一步都有明确的数学公式。

---

### 第一步：构造训练样本对（线性插值路径）

对于每一对训练数据，我们有：

- **条件**：$c = z_{ppg}$（PPG 的潜在表示）
- **目标**：$y = z_{ecg}$（ECG 的潜在表示）
- **噪声**：$z \sim \mathcal{N}(0, I)$（从标准高斯分布采样的纯噪声）

然后，在时间 $t \in [0, 1]$ 上构造一条**线性插值路径**：

$$x_t = (1 - t)z + t y$$

(Fang 等, 2026)

**直观理解**：
- $t=0$：$x_0 = z$，纯高斯噪声
- $t=1$：$x_1 = y$，目标 ECG 潜在
- $t=0.5$：$x_{0.5} = 0.5z + 0.5y$，噪声和目标的混合

这条路径的**最优漂移（理想速度）**是常数：

$$v^*(x_t, t, c) = y - z$$

(Fang 等, 2026)

也就是说，从 $t=0$ 到 $t=1$，每一步都应该朝着 $(y - z)$ 这个方向走。

---

### 第二步：训练速度场网络 $v_\theta$（监督回归）

我们用一个神经网络 $v_\theta$（Transformer 骨干网络）来**逼近这个理想速度**。

训练目标是最小化均方误差：

$$\mathcal{L}(\theta) = \mathbb{E}_{z \sim \mathcal{N}(0,I),\; y,\; t \sim U(0,1)} \left\| v_\theta(x_t, t, c) - (y - z) \right\|_2^2$$

(Fang 等, 2026)

**训练流程（每个 batch 的步骤）**：

1. 从数据集中采样一对 $(z_{ppg}, z_{ecg})$ → 得到 $c = z_{ppg}$, $y = z_{ecg}$
2. 从标准高斯采样 $z \sim \mathcal{N}(0, I)$
3. 从均匀分布采样 $t \sim U(0, 1)$
4. 计算插值点：$x_t = (1-t)z + t \cdot y$
5. 计算监督信号（理想速度）：$\text{target} = y - z$
6. 前向传播：$v_{\text{pred}} = v_\theta(x_t, t, c)$
7. 计算损失：$\mathcal{L} = \|v_{\text{pred}} - \text{target}\|^2$
8. 反向传播更新 $v_\theta$ 的参数

**关键点**：$v_\theta$ 的输入有三个：
- $x_t$：当前插值位置（噪声和目标的混合）
- $t$：当前时间步
- $c = z_{ppg}$：PPG 潜在（通过交叉注意力注入）

输出是一个向量，告诉系统"从这个位置应该往哪个方向走"。

---

### 第三步：推理时用 ODE 求解器生成

训练完成后，推理时从纯噪声开始，沿着学到的速度场走 T 步：

$$\frac{dx_t}{dt} = v_\theta(x_t, t, c), \quad t \in [0, 1]$$

(Fang 等, 2026)

用显式欧拉法离散求解：

$$x_{k+1} = x_k + \Delta t \cdot v_\theta(x_k, t_k, c), \quad k = 0, 1, \ldots, T-1$$

(Fang 等, 2026)

其中 $\Delta t = 1/T$，$t_k = k \cdot \Delta t$。

最终 $x_T$ 就是生成的 ECG 潜在表示，通过冻结的 ECG 解码器重建为波形。

---

### 一个具体的数值例子

假设潜在空间是 2 维（便于可视化）：

**训练时**：
```
z = [0.5, -0.3]  (高斯噪声)
y = [1.2, 0.8]   (ECG 潜在)
t = 0.3
x_t = 0.7×[0.5,-0.3] + 0.3×[1.2,0.8] = [0.71, -0.01]
target = y - z = [0.7, 1.1]
v_θ([0.71,-0.01], 0.3, c) 应该输出 ≈ [0.7, 1.1]
```

**推理时**：
```
从 z = [0.5, -0.3] 开始
第1步：x₁ = [0.5,-0.3] + 0.1 × v_θ([0.5,-0.3], 0, c)
第2步：x₂ = x₁ + 0.1 × v_θ(x₁, 0.1, c)
...
第10步：x₁₀ ≈ [1.2, 0.8] → 解码为 ECG 波形
```

### 网络架构：$v_\theta$ 具体长什么样？

> To estimate $v_\theta$, we use a Transformer backbone with hierarchical cross-modal conditioning. The ECG latent $y$ is projected into a compact token sequence via a 1D convolutional embedder... The PPG latent $c$ conditions generation through cross-attention in every Transformer block, with $c$ as key–value and the evolving ECG tokens as queries.

(Fang 等, 2026)

- **输入处理**：ECG 潜在 $y$ 通过 1D 卷积嵌入为 token 序列
- **条件注入**：PPG 潜在 $c$ 通过每个 Transformer 块的**交叉注意力**作为 key-value 注入
- **时间编码**：$t$ 通过位置编码或时间嵌入输入

---

### 为什么 Stage 1 的对齐让这个学习更容易？

论文给出了形式化证明：

**条件分散度**：
$$\bar{\kappa} = \mathbb{E}_c\left[\text{tr}\left(\text{Cov}(y | c)\right)\right]$$

Stage 1 的三层次对齐使 $\bar{\kappa} \to 0$（给定 PPG 潜在时，ECG 潜在的不确定性很小）。

**不可约贝叶斯风险**：
$$R_{\min} \leq \bar{\kappa}$$

(Fang 等, 2026)

**翻译成人话**：Stage 1 把 $z_{ppg}$ 和 $z_{ecg}$ 在潜在空间中拉近后，$y - z$ 这个理想速度的方差变小了，$v_\theta$ 更容易学准，而且学出来的向量场曲率更低，5-10 步 ODE 就能精确求解。

---

### 一句话总结

> **速度场 $v_\theta$ 是通过"监督回归"学出来的**：对每个训练样本，构造从高斯噪声 $z$ 到 ECG 潜在 $y$ 的直线路径，让 $v_\theta(x_t, t, c)$ 去预测路径上每个点的瞬时方向 $(y - z)$。Stage 1 的对齐让 $y$ 和 $c$ 在空间中靠近，降低了这个回归任务的难度。
