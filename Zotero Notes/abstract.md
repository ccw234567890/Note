
# $π_{0.7}$: a Steerable Generalist Robotic Foundation Model with Emergent Capabilities

## 核心概念讲解

### 一、总体定位

**基础模型（Foundation Model）**：在大规模、多样化数据上预训练的通用模型，可直接用于多种下游任务而无需针对每个任务重新训练。类比 GPT 是语言的基础模型，π₀.₇ 是**机器人操作的基础模型**。

**开箱即用（Out-of-the-box）**：模型**不需要任何任务特定的微调或后训练**，直接部署到新环境就能工作。这是 π₀.₇ 的核心突破——之前的机器人模型通常需要针对每个新任务、新环境重新收集数据并微调。

**看不见的环境（Unseen Environments）**：模型在训练时从未见过的物理场景。论文中测试了 4 个未见厨房 + 2 个未见卧室，模型能直接在这些新环境中执行任务。

**多阶段任务（Multi-stage Tasks）**：需要按顺序完成多个子步骤的复杂任务。例如"清理厨房"可能包含：打开冰箱 → 取出食材 → 关上冰箱 → 走到台面 → 切菜 → 放入碗中。

### 二、零样本跨本体迁移

**零样本跨本体迁移（Zero-shot Cross-embodiment Generalization）**：π₀.₇ 最令人惊叹的涌现能力之一。

- **本体（Embodiment）**：机器人的物理形态——臂的数量、关节配置、尺寸、重量等
- **跨本体迁移**：将在一种机器人上训练的技能转移到另一种不同形态的机器人上
- **零样本**：目标机器人**从未收集过任何相关任务的数据**

**具体实验**：在 BiPi（轻量双臂机器人）上训练的衣物折叠技能，直接迁移到 **UR5e**（更大、更重的工业级双臂机器人）上。UR5e 从未见过衣物折叠数据，但 π₀.₇ 能直接执行，甚至**发现了更适合 UR5e 的新策略**（单臂操作 vs 源机器人的双臂操作）。

**RL 微调模型（RL-finetuned Specialist）**：π₀*.6 模型——对每个任务单独使用强化学习（RL）进行微调的专家版本。RL 微调通常需要大量在线交互和精心设计的奖励函数，成本极高。

**关键对比**：π₀.₇ **一个通用模型**，在多个高难度任务上的性能，与**每个任务单独 RL 微调的专家模型**相当，甚至在衣物折叠、纸箱折叠等任务上**吞吐量更高**（执行速度更快）。

### 三、多样化上下文条件

**多样化上下文条件（Diverse Contextual Conditioning）**：π₀.₇ 最核心的方法创新。传统机器人策略的输入通常只是当前观测 + 任务指令。π₀.₇ 的输入**远不止这些**，它接收一个丰富的**上下文提示（Context Prompt）** $C_t$，包含多种条件信息。

**条件反射（Conditioning）**：给模型提供额外的输入信息来"引导"其输出。就像给一个人说"用慢速、精确的方式折叠这件衬衫"——同样的任务，不同的条件导致不同的行为。

#### (a) 子任务指令（Subtask Instructions）

除了整体任务描述，还包含**中间语义子任务** $l̂_t$，如：
- 整体任务："清理厨房"
- 子任务序列："打开冰箱门" → "取出蔬菜" → "关上冰箱" → "走到台面" → "切菜"

**意义**：使模型能执行**多阶段、长时任务**，高层策略或人类可以实时提供子任务指令（称为**语言教练**）。

#### (b) 子目标图像（Subgoal Images）

多视角的**期望未来状态图像** $g_t = [G_t^1, ..., G_t^n]$，由轻量级**世界模型**（基于 BAGEL 14B 图像生成模型）实时生成。

**例子**：如果任务是"把杯子放在托盘上"，子目标图像可能显示"手已经抓住杯子"的状态。

**训练时**：混合使用真实未来帧（25% 段末帧 + 75% 0-4秒随机帧）和世界模型生成的图像。

**推理时**：每 4 秒或子任务变化时刷新子目标图像。

#### (c) 片段元数据（Episode Metadata）

| 元数据字段 | 含义 | 示例 |
|-----------|------|------|
| **速度（Speed）** | 片段长度（时间步数），离散化为 500 步区间 | `Speed: 1`（快速）或 `Speed: 4`（慢速） |
| **质量（Quality）** | 1-5 分的任务执行质量评分 | `Quality: 5`（专家级）或 `Quality: 2`（较差） |
| **错误标记（Mistake）** | 是否在动作段中犯错 | `Mistake: false`（无错误）或 `Mistake: true`（有错误） |

**关键作用**：这些元数据使 π₀.₇ 能够从**失败和次优数据**中学习。推理时设置 `Quality: 5, Mistake: false`，模型就会输出专家级别的行为——这本质上是一种**知识蒸馏**，将 RL 专家的能力蒸馏到通用模型中。

#### (d) 控制模式（Control Mode）

同时训练关节级（joint）和末端执行器（end-effector）两种控制模式，推理时可根据任务选择。

### 四、多样化数据

π₀.₇ 的训练数据包含：

| 数据来源 | 质量水平 | 是否被传统方法使用 |
|---------|---------|-----------------|
| 人类演示数据 | 高 | ✅ 是 |
| 自主数据（成功） | 中高 | ✅ 是 |
| **自主数据（失败）** | **低** | ❌ 传统方法丢弃 |
| **π₀*.6 RL 专家 rollout** | **混合** | ❌ 传统方法丢弃 |
| 人类干预数据 | 中 | ❌ 通常丢弃 |
| 开源机器人数据 | 混合 | ✅ 部分使用 |
| 人类视频（非机器人） | 无动作标签 | ❌ 通常丢弃 |
| 网络数据 | 无动作标签 | ❌ 通常丢弃 |

**元数据的关键作用**：传统方法只能使用高质量数据，因为低质量数据会"污染"模型。但 π₀.₇ 通过元数据标注，让模型**学会区分好坏**——训练时看到各种质量的数据，推理时指定高质量条件，模型就会输出高质量行为。

**消融实验证明**：
- 移除元数据（π₀.₇ no metadata）→ 所有任务性能显著下降
- 移除 RL 专家数据（π₀.₇ no eval data）→ 所有任务性能显著下降
- 证明**元数据 + 多样化数据**是 π₀.₇ 成功的关键

### 五、核心概念关系图

```
π₀.₇ 的核心创新
│
├── 开箱即用性能
│   ├── 无需任务特定微调
│   ├── 匹配 RL 专家性能
│   └── 吞吐量更高
│
├── 涌现能力
│   ├── 零样本跨本体迁移（BiPi → UR5e 折叠衣物）
│   ├── 指令跟随（未见环境、反向指令）
│   └── 组合泛化（语言教练学习新任务）
│
└── 核心方法：多样化上下文条件
    │
    ├── 语言命令（做什么）
    ├── 子任务指令（怎么做——步骤分解）
    ├── 子目标图像（怎么做——视觉目标）
    └── 元数据（怎么做——质量/速度/错误）
        ├── 速度：快慢
        ├── 质量：1-5 分
        └── 错误标记：有无错误
            │
            └── 使模型能从混合质量数据中学习
                ├── 失败数据 → 学会避免错误
                ├── RL 专家数据 → 蒸馏专家能力
                └── 人类视频 → 学习通用视觉知识
```
