
# 扩散模型

## 概述

扩散模型（Diffusion Models）是一类**生成模型**，其核心思想是：先通过一个**前向过程**逐步向数据中添加噪声，直到数据完全变成纯随机噪声；然后学习一个**反向过程**，从纯噪声中逐步去噪，恢复出原始数据。扩散模型在图像生成领域取得了巨大成功（如 Stable Diffusion、DALL·E、Imagen），近年来也被引入机器人控制领域用于生成连续动作。

π₀ 论文中使用的 **Flow Matching（流匹配）** 是扩散模型的一种变体，论文中明确将其描述为 "a variant of diffusion"（扩散模型的一种变体）：

> In contrast, our model employs a novel design that fine-tunes a VLM to produce actions via flow matching, a variant of diffusion.
(Black 等, 2026)

## 一、扩散模型的基本原理

### 1.1 前向扩散过程（Forward Diffusion Process）

前向过程是一个**马尔可夫链**，逐步向数据 $x_0$ 添加高斯噪声，经过 $T$ 步后，数据完全变成标准高斯噪声 $x_T \sim \mathcal{N}(0, I)$：

$$q(x_t | x_{t-1}) = \mathcal{N}\left(x_t; \sqrt{1-\beta_t}\,x_{t-1},\; \beta_t I\right)$$

其中 $\beta_t$ 是噪声调度（noise schedule），控制每一步添加噪声的强度。

一个重要的性质是：**可以直接从 $x_0$ 一步跳到任意时间步 $t$**，无需逐步迭代：

$$q(x_t | x_0) = \mathcal{N}\left(x_t; \sqrt{\bar{\alpha}_t}\,x_0,\; (1-\bar{\alpha}_t)I\right)$$

其中 $\bar{\alpha}_t = \prod_{i=1}^t (1-\beta_i)$。这等价于：

$$x_t = \sqrt{\bar{\alpha}_t}\,x_0 + \sqrt{1-\bar{\alpha}_t}\,\varepsilon,\quad \varepsilon \sim \mathcal{N}(0, I)$$

### 1.2 反向去噪过程（Reverse Denoising Process）

反向过程的目标是学习一个神经网络 $\varepsilon_\theta(x_t, t)$，预测当前噪声 $x_t$ 中所含的噪声 $\varepsilon$，从而逐步去除噪声恢复数据：

$$p_\theta(x_{t-1} | x_t) = \mathcal{N}\left(x_{t-1}; \mu_\theta(x_t, t),\; \Sigma_\theta(x_t, t)\right)$$

训练时最小化预测噪声与真实噪声之间的均方误差：

$$\mathcal{L}_{\text{simple}}(\theta) = \mathbb{E}_{t, x_0, \varepsilon} \left\| \varepsilon - \varepsilon_\theta(x_t, t) \right\|^2$$

### 1.3 推理（采样）过程

推理时从纯噪声 $x_T \sim \mathcal{N}(0, I)$ 出发，通过训练好的网络 $\varepsilon_\theta$ 逐步去噪，经过 $T$ 步迭代后得到生成的样本 $x_0$。每一步的更新公式为：

$$x_{t-1} = \frac{1}{\sqrt{\alpha_t}}\left(x_t - \frac{1-\alpha_t}{\sqrt{1-\bar{\alpha}_t}}\,\varepsilon_\theta(x_t, t)\right) + \sigma_t z$$

其中 $z \sim \mathcal{N}(0, I)$ 是随机项，$\sigma_t$ 是噪声方差。

## 二、扩散模型的关键特性

### 2.1 连续分布建模

与自回归模型（将输出离散化为有限类别）不同，扩散模型直接在**连续空间**中建模数据的分布。这使得它特别适合处理机器人动作这类连续信号，避免了离散化带来的量化误差。

### 2.2 多模态分布

扩散模型能够建模**多模态分布**（multimodal distribution），即同一个输入可以对应多个合理的输出。对于机器人控制来说，这意味着面对同一场景可以有多种不同的动作方案，模型不会坍缩到单一"平均动作"。

### 2.3 迭代精化

扩散模型通过多步迭代逐步生成结果，每一步都在前一步的基础上进行精化。这种"从粗到细"的生成方式使得模型能够捕捉高频细节。

## 三、从扩散模型到流匹配（Flow Matching）

π₀ 使用的 **Flow Matching** 是扩散模型的一种变体，两者共享相同的核心思想——从噪声到数据的逐步转换，但在具体实现上有几个关键区别：

### 3.1 连续时间框架

| 特性 | 经典扩散模型 | 流匹配 |
|------|-------------|--------|
| 时间步 | 离散的 $t=1,2,\dots,T$ | 连续的 $\tau \in [0, 1]$ |
| 噪声路径 | 方差调度 $\beta_t$ | 线性高斯路径 |
| 训练目标 | 预测噪声 $\varepsilon$ | 预测向量场 $v_\theta$ |
| 采样方式 | 离散去噪步 | 欧拉积分 |

### 3.2 线性高斯概率路径

流匹配使用**线性高斯概率路径**（Linear Gaussian Probability Path）来定义噪声与数据之间的插值关系：

$$q(A_t^\tau | A_t) = \mathcal{N}\left(\tau A_t,\; (1-\tau)I\right)$$

这意味着带噪动作可以直接写为：

$$A_t^\tau = \tau A_t + (1-\tau)\varepsilon,\quad \varepsilon \sim \mathcal{N}(0, I)$$

其中 $\tau=0$ 时 $A_t^0 = \varepsilon$（纯噪声），$\tau=1$ 时 $A_t^1 = A_t$（干净数据）。

### 3.3 目标向量场

从线性高斯路径可以解析推导出**条件向量场**（conditional vector field），它描述了在位置 $A_t^\tau$ 处粒子应该朝哪个方向运动：

$$u(A_t^\tau | A_t) = A_t - \varepsilon$$

### 3.4 流匹配损失函数

流匹配训练神经网络 $v_\theta$ 去拟合这个向量场：

$$\mathcal{L}_\tau(\theta) = \mathbb{E}_{p(A_t|o_t), q(A_t^\tau|A_t)} \left\| v_\theta(A_t^\tau, o_t) - u(A_t^\tau|A_t) \right\|^2$$

代入 $u$ 的解析式后：

$$\mathcal{L}_\tau(\theta) = \mathbb{E}_{p(A_t|o_t), \varepsilon \sim \mathcal{N}(0,I)} \left\| v_\theta(\tau A_t + (1-\tau)\varepsilon, o_t) - (A_t - \varepsilon) \right\|^2$$

### 3.5 推理：欧拉积分

推理时从纯噪声 $A_t^0 \sim \mathcal{N}(0, I)$ 出发，通过欧拉积分逐步求解向量场：

$$A_t^{\tau+\delta} = A_t^\tau + \delta \cdot v_\theta(A_t^\tau, o_t)$$

π₀ 使用 $\delta=0.1$（10 步积分）即可生成高质量动作。

## 四、扩散模型 vs 流匹配：类比

| 概念 | 经典扩散模型 | 流匹配 |
|------|-------------|--------|
| 前向过程 | $x_t = \sqrt{\bar{\alpha}_t}x_0 + \sqrt{1-\bar{\alpha}_t}\varepsilon$ | $A_t^\tau = \tau A_t + (1-\tau)\varepsilon$ |
| 网络预测 | 噪声 $\varepsilon_\theta(x_t, t)$ | 向量场 $v_\theta(A_t^\tau, o_t)$ |
| 损失函数 | $\|\varepsilon - \varepsilon_\theta\|^2$ | $\|(A_t-\varepsilon) - v_\theta\|^2$ |
| 采样方式 | 离散去噪 $x_{t-1} = f(x_t, \varepsilon_\theta)$ | 欧拉积分 $A_t^{\tau+\delta} = A_t^\tau + \delta v_\theta$ |
| 步数 | 通常 50-1000 步 | 通常 10-50 步 |

流匹配的优势在于：**更少的采样步数**（10 步 vs 扩散模型的 50-1000 步），这对于需要实时控制的机器人应用至关重要。

## 五、扩散模型在机器人控制中的应用

### 5.1 为什么扩散模型适合机器人控制？

1. **连续动作空间**：机器人动作（关节角度、末端速度）是连续值，扩散模型天然适合连续分布建模
2. **多模态动作**：同一场景下可以有多种合理的动作方案，扩散模型能保留这种多模态性
3. **高频控制**：流匹配仅需 10 步积分，可实现 50 Hz 的实时控制频率
4. **动作分块**：扩散模型可以一次性生成整个动作块 $A_t = [a_t, a_{t+1}, \dots, a_{t+H-1}]$，实现时间维度的并行预测

### 5.2 与自回归方法的对比

传统 VLA 模型（如 OpenVLA）将动作离散化为文本令牌，使用自回归方式逐个预测：

> OpenVLA struggles on these tasks because its autoregressive discretization architecture does not support action chunks.
(Black 等, 2026)

而扩散/流匹配方法直接在连续空间中生成整个动作块，避免了离散化损失，且支持并行预测。

## 六、电子信息工程视角

从信号处理的角度理解扩散模型：

### 6.1 前向过程 = 加性白噪声信道

前向扩散过程 $x_t = \sqrt{\bar{\alpha}_t}x_0 + \sqrt{1-\bar{\alpha}_t}\varepsilon$ 类似于通信系统中的**加性白高斯噪声（AWGN）信道**，其中信噪比 SNR $= \bar{\alpha}_t/(1-\bar{\alpha}_t)$ 随 $t$ 增加而逐渐降低。

### 6.2 反向过程 = 迭代去噪均衡

反向去噪过程类似于数字接收机中的**迭代均衡器**（iterative equalizer），每一步利用神经网络估计当前噪声，然后从接收信号中减去噪声分量，逐步恢复原始信号。

### 6.3 流匹配 = 连续时间最优接收机

流匹配将离散时间步扩展为连续时间 $\tau \in [0,1]$，相当于将接收机从离散时间域推广到连续时间域。欧拉积分 $A_t^{\tau+\delta} = A_t^\tau + \delta v_\theta$ 类似于**连续时间系统的数值求解**，每一步都在当前估计的基础上沿梯度方向前进。

### 6.4 动作分块 = 数据包传输

动作分块 $A_t = [a_t, a_{t+1}, \dots, a_{t+H-1}]$ 类似于通信中的**数据包**（packet）传输——将连续的动作流分块打包，一次性生成并发送，提高了传输效率（推理频率降低到 $1/H$）。

---

## 总结

扩散模型是一类通过"先加噪、再去噪"来生成数据的生成模型。流匹配作为扩散模型的一种变体，使用连续时间框架和线性高斯概率路径，仅需少量积分步即可生成高质量样本。在 π₀ 中，流匹配被用于生成连续机器人动作，结合动作分块架构实现了 50 Hz 的高频灵巧控制，克服了传统自回归 VLA 模型在连续动作建模上的局限性。
